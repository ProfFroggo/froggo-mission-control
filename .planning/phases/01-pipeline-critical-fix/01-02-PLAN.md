<task type="auto">
  <name>Task 1: Add periodic stuck-in-progress detection to review cycle</name>
  <files>src/lib/claraReviewCron.ts</files>
  <action>
In the `runReviewCycle()` function, after the post-work review pass, add a stuck task recovery pass:

1. Query tasks that have been in `in-progress` for more than 30 minutes with no recent activity:
   ```sql
   SELECT id, title, assignedTo, updatedAt FROM tasks
   WHERE status = 'in-progress'
     AND updatedAt < ?  -- 30 min ago
     AND id NOT IN (SELECT DISTINCT taskId FROM task_activity WHERE timestamp > ?)  -- no recent activity
   ```

2. For each stuck task:
   - Check if the agent process is still running (look for active dispatches in the module)
   - If no active process: re-dispatch with message "Task was stuck in-progress with no running agent. Resuming work."
   - Log activity: 'stuck-recovery' with message
   - Limit to 3 recovery attempts (track via a stuck_recovery_count or reuse claraReviewCount)

3. Don't recover tasks that were JUST rejected by Clara (check lastClaraReviewAt — if within 10 minutes, skip — the re-dispatch from Plan 01-01 will handle it)

4. Log: `[clara-review-cron] Recovered N stuck in-progress task(s)`
  </action>

Run `npx tsc --noEmit` — zero errors.
  </verify>
  <done>
Tasks stuck in in-progress for >30 minutes with no activity are automatically re-dispatched. Recovery is logged. Limit of 3 attempts prevents infinite loops.
  </done>
