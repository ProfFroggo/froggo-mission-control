# Plan 01-01: Remove Electron, Install Next.js, Update Configs

## Objective

Strip all Electron dependencies from the project, install Next.js App Router, and update `next.config.js`, `tsconfig.json`, and `package.json` scripts so the project can build as a web app.

## Context

- Repo: `~/git/mission-control-nextjs`, branch: `migration/nextjs-claude-code`
- Current stack: Vite + React 18 + TypeScript + Tailwind + Electron
- All Electron-specific deps must be removed; `better-sqlite3` is kept (works in Next.js API routes)
- Current scripts: `vite`, `electron:compile`, etc. — all replaced with `next` equivalents
- Spec: `fork-merge.md` §1.1–1.6

## Tasks

### Task 1: Install Next.js and remove Electron deps

```bash
cd ~/git/mission-control-nextjs

# Install Next.js
npm install next@latest
npm install -D @types/node

# Remove Electron-specific packages
npm uninstall electron electron-builder electron-is-dev electron-store electron-updater @electron/fuses electron-rebuild

# Also remove vite and its plugins (Next.js replaces vite as bundler)
# Keep vitest for testing — it works independently
npm uninstall vite @vitejs/plugin-react rollup-plugin-visualizer terser concurrently wait-on
```

**Verify:** `package.json` no longer has `electron`, `electron-builder`, `vite` in deps/devDeps.

### Task 2: Create next.config.js

Create `next.config.js` at project root:

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    serverComponentsExternalPackages: ['better-sqlite3'],
  },
  webpack: (config) => {
    config.resolve.alias = {
      ...config.resolve.alias,
      '@': require('path').resolve(__dirname, 'src'),
    };
    return config;
  },
};

module.exports = nextConfig;
```

### Task 3: Update tsconfig.json

Replace content of `tsconfig.json` with Next.js-compatible config. Key changes from current:
- `target`: ES2020 → ES2017
- `lib`: add `esnext`, remove `DOM.Iterable` restriction
- `module`: ESNext → esnext
- `jsx`: react-jsx → preserve (Next.js handles JSX transform)
- `plugins`: add `{ "name": "next" }`
- `include`: add `next-env.d.ts`, `.next/types/**/*.ts`; remove direct `src` reference (use `**/*.ts`)
- `exclude`: add `electron`, `dist`, `dist-electron`, `release`
- Remove `allowImportingTsExtensions` (not compatible with Next.js bundler mode)
- Remove `references` (no longer using tsconfig.node.json)
- Remove `noUnusedLocals` and `noUnusedParameters` (too strict for migration — existing Electron code has many unused imports)

New `tsconfig.json`:

```json
{
  "compilerOptions": {
    "target": "ES2017",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "plugins": [{ "name": "next" }],
    "paths": {
      "@/*": ["./src/*"]
    },
    "baseUrl": "."
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules", "electron", "dist", "dist-electron", "release"]
}
```

### Task 4: Update package.json scripts

Replace the electron/vite scripts with Next.js equivalents. The `postinstall` script that ran `electron-builder install-app-deps` must also be removed.

New scripts section:

```json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "next lint",
    "test": "vitest",
    "test:ui": "vitest --ui",
    "test:run": "vitest run",
    "test:coverage": "vitest run --coverage",
    "test:e2e": "playwright test"
  }
}
```

Remove the `postinstall` hook entirely (it was `electron-builder install-app-deps`).

### Task 5: Remove Electron source files from compilation

The `electron/` directory contains TypeScript files that will no longer compile under the new tsconfig (they import Electron APIs). They stay in the repo for reference but must be excluded. The new `tsconfig.json` already excludes the `electron` directory.

Also check if `tsconfig.electron.json` and `tsconfig.node.json` reference anything that would cause issues — they can remain as-is since they're no longer referenced.

### Task 6: Commit

```bash
cd ~/git/mission-control-nextjs

git add -f .planning/phases/01-electron-strip-nextjs-scaffold/

```
