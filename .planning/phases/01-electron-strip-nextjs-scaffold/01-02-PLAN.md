### Task 2: Create app/layout.tsx

This is the root layout — wraps every page. Import global CSS here.

Create `app/layout.tsx`:

```tsx
import type { Metadata } from 'next';
import '../src/index.css';
import '../src/accessibility.css';

export const metadata: Metadata = {
  title: 'Froggo Dashboard',
  description: 'Multi-agent orchestration platform',
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en" className="dark">
      <body>{children}</body>
    </html>
  );
}
```

**Note:** Import both `src/index.css` and `src/accessibility.css` here since `src/main.tsx` imports both. The other CSS files (`component-patterns.css`, `dashboard-redesign.css`, etc.) are likely imported by individual components — leave those as-is.

### Task 4: Create next-env.d.ts

```typescript
// NOTE: This file should not be edited

```

### Task 5: Handle TypeScript compilation issues

The existing codebase has some patterns that may conflict with Next.js strict compilation. Check for issues:

```bash
cd ~/git/froggo-nextjs
npx tsc --noEmit 2>&1 | head -50
```

Common issues to fix:
- **`allowImportingTsExtensions`** was in old tsconfig but removed — any `import './foo.tsx'` with explicit extensions may need the extension removed
- **`window.clawdbot`** references — `src/lib/gateway.ts` and components use this. Add a type declaration to `src/vite-env.d.ts` or create `src/global.d.ts`:
  ```typescript
  // src/global.d.ts
  declare global {
    interface Window {
      clawdbot?: any;
    }
  }
  export {};
  ```
- **Electron-specific imports** — any `src/` file that imports from `electron` (e.g., `ipcRenderer`) will error. These are in the preload and some utility files. Add `// @ts-ignore` or move them inside `if (typeof window !== 'undefined' && window.electron)` guards temporarily.

Fix TypeScript errors that block compilation. Do NOT fix Electron-specific business logic — just make the TypeScript compiler happy enough to continue.
