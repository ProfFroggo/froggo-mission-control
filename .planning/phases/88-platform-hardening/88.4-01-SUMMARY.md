# Phase 88.4 ‚Äî Real-Time Architecture

## What was done

### Plan 01: SSE‚ÜíStore bridge + kill redundant polls

**Central SSE‚ÜíStore bridge** (`src/hooks/useSSEStoreSync.ts`):
- `task.created` ‚Üí reloads tasks from DB
- `task.updated` ‚Üí applies partial delta (status, assignedTo, priority) via `patchTaskLocal` ‚Äî no full refetch
- `agent.status` / `agent.updated` ‚Üí updates agent status in store
- `budget.dispatch_blocked` ‚Üí adds system activity
- Mounted once in `App.tsx`

**Polling reduction** (all intervals increased, SSE is primary):
- Kanban task polling: 60s ‚Üí 120s (removed redundant SSE handlers that triggered full refetch)
- useAgents polling: 15s ‚Üí 120s
- useNotifications polling: 10s ‚Üí 60s
- ApprovalQueuePanel polling: 20s ‚Üí 120s

### Plan 02: EventBus reconnection + connection status + missing SSE events

**EventBus reconnection fix** (`src/lib/useEventBus.ts`):
- Added `onopen` handler to track connected state
- `onerror`: if readyState is CLOSED (auto-reconnect gave up), schedules manual reconnect in 5s
- Connection state tracked via module-level variable + listener pattern

**Connection status indicator** (`src/components/TopBar.tsx`):
- Added `useSSEConnectionState()` hook (uses `useSyncExternalStore`)
- Amber pulsing dot when reconnecting, red dot when disconnected
- Hidden when connected (no noise in happy path)

**Missing SSE events**:
- Added `emitSSEEvent('approval.created', ...)` to POST /api/approvals
- Added `emitSSEEvent('approval.updated', ...)` to PATCH /api/approvals
- ApprovalQueuePanel now listens to both SSE events for immediate refresh
- Added `budget.dispatch_blocked`, `dispatch.auto_routed`, `approval.created`, `approval.updated` to EVENT_TYPES
