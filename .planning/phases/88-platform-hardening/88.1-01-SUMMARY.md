# Phase 88.1 — Dispatch Engine Critical Fixes

## What was done

### Task 1: Set-based concurrency tracking
- Replaced `let activeDispatches = 0` counter with `const activeDispatchSet = new Set<string>()`
- `getActiveDispatchCount()` returns `activeDispatchSet.size`
- All ~12 increment/decrement sites converted to `activeDispatchSet.add(taskId)` / `activeDispatchSet.delete(taskId)`
- Double-decrement race eliminated: `Set.delete()` on missing key is a no-op

### Task 2: Agent-scoped session keys + double preemption guard
- **Session keys**: Changed from `'task:' + taskId` to `'task:' + taskId + ':' + agentId` across all 5 locations (load, persist, stale_session delete, context_overflow delete, oom_killed delete)
- `loadTaskSession()` and `persistTaskSession()` now take `agentId` parameter; all call sites updated
- **Double preemption guard**: Added `preemptionUsed` flag — first successful `preemptForP0()` sets it, second call site gated by `!preemptionUsed`

### Task 3: Timestamp standardization
- Changed `agents.created_at` from `DEFAULT (unixepoch())` to `DEFAULT (unixepoch() * 1000)`
- Also fixed the ALTER TABLE migration statement
- Added comment: `-- All timestamps in milliseconds (Date.now() compatible)`
- Zero bare `unixepoch()` defaults remain
