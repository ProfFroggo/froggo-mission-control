<task type="auto">
  <name>Task 1: Replace activeDispatches counter with Set-based tracking</name>
  <files>src/lib/taskDispatcher.ts</files>
  <action>
  Replace the `let activeDispatches = 0` counter and `MAX_CONCURRENT_DISPATCHES` guard with a `Set<string>` called `activeDispatchSet` that tracks taskIds.

  1. Replace declaration (around line 1157):
     - Remove: `let activeDispatches = 0`
     - Add: `const activeDispatchSet = new Set<string>()`
  2. Replace `getActiveDispatchCount()` to return `activeDispatchSet.size`
  3. Replace the concurrency check (`activeDispatches >= MAX_CONCURRENT_DISPATCHES`) with `activeDispatchSet.size >= MAX_CONCURRENT_DISPATCHES`
  4. Replace increment (`activeDispatches++`) with `activeDispatchSet.add(taskId)`
  5. Replace ALL decrements (`activeDispatches = Math.max(0, activeDispatches - 1)`) with `activeDispatchSet.delete(taskId)` — there are ~6 locations:
     - Success path close handler
     - preemptForP0() kill path
     - Rate limit exhaustion path
     - Auth error path
     - Process timeout handler
     - Any other decrement location
  6. In preemptForP0(), use `activeDispatchSet.delete(preemptable.id)` (the PREEMPTED task's id, not the P0 task's id)

  This eliminates the double-decrement race: if preemption deletes the taskId from the set, the close handler's delete is a no-op (Set.delete on missing key returns false, no crash).

  Do NOT change the logic flow — only replace the counter mechanism. Keep MAX_CONCURRENT_DISPATCHES as 2.
  </action>
  <verify>Grep for `activeDispatches` — should only appear in the old variable name pattern. All references should now use `activeDispatchSet`. `npm run build:verify` passes.</verify>
  <done>No bare counter. Set-based tracking prevents double-decrement. getActiveDispatchCount returns Set size. Build passes.</done>
</task>
