# Phase 88.2 — Data Integrity & Safety

## What was done

### Task 1: Budgets table PK + unique constraint
- Added `id TEXT PRIMARY KEY DEFAULT (lower(hex(randomblob(8))))` to budgets table
- Added `UNIQUE(agentId, period, name)` constraint to prevent duplicate budget entries

### Task 2: Inbox spawn guard
- Added `MAX_INBOX_SPAWNS = 2` concurrency limit with `activeInboxSpawns` Set tracking
- Added 5-minute SIGTERM timeout with 5s SIGKILL escalation
- Cleanup on exit/error events
- Logs warning when spawn limit reached

### Task 3: Recurrence transaction
- Wrapped check-then-insert for recurring task spawn in `db.transaction()` — prevents duplicate recurrence spawns from concurrent requests

### Task 4: Multi-step mutations wrapped in transactions
- **convert-to-task**: INSERT task + UPDATE inbox status now atomic
- **project creation**: INSERT project + INSERT members + INSERT chat room now atomic
- Removed redundant duplicate `mkdirSync` call in projects route
- Filesystem operations (non-transactional) moved outside DB transaction
