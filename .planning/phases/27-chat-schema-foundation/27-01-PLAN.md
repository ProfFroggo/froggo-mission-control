In `initDb()`, find the base CREATE TABLE statements for chat_rooms (around line 297) and chat_room_messages (around line 304). Expand them to include ALL columns that are currently added via ALTER TABLE migrations.

For `chat_rooms`, the complete schema should be:
```sql
CREATE TABLE IF NOT EXISTS chat_rooms (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  topic TEXT,
  agents TEXT DEFAULT '[]',
  sessionKeys TEXT DEFAULT '{}',
  updatedAt INTEGER DEFAULT 0,
  project_id TEXT REFERENCES projects(id) ON DELETE SET NULL,
  pinnedMessageId TEXT,
  description TEXT,
  createdAt INTEGER DEFAULT (unixepoch() * 1000)
);
```

For `chat_room_messages`, the complete schema should be:
```sql
CREATE TABLE IF NOT EXISTS chat_room_messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  roomId TEXT NOT NULL,
  agentId TEXT NOT NULL,
  content TEXT NOT NULL,
  replyTo INTEGER,
  timestamp INTEGER NOT NULL DEFAULT (unixepoch() * 1000),
  role TEXT DEFAULT 'agent',
  mentionedAgents TEXT DEFAULT '[]',
  messageId TEXT,
  parentId TEXT,
  FOREIGN KEY (roomId) REFERENCES chat_rooms(id),
  FOREIGN KEY (replyTo) REFERENCES chat_room_messages(id)
);
```

IMPORTANT: Do NOT remove the corresponding entries from `columnMigrations`. Existing databases that were created before these columns were added still need the ALTER TABLE migrations to run on startup. The base CREATE TABLE and columnMigrations can safely coexist — IF NOT EXISTS on CREATE TABLE means fresh DBs get the full schema, and ALTER TABLE try/catch means existing DBs silently skip already-existing columns.

**Bug 1 — rooms sort order** (`app/api/chat-rooms/route.ts` GET handler):
Change the query from:
```sql
ORDER BY r.createdAt ASC
```
to:
```sql
ORDER BY COALESCE(r.updatedAt, r.createdAt) DESC
```
This makes rooms sort by most recently active first (matching chat app conventions).

**Bug 2 — duplicate PATCH field** (`app/api/chat-rooms/[roomId]/route.ts` PATCH handler):
Lines 30-31 both push `pinnedMessageId` to the updates array. Remove the duplicate at line 31 entirely. Only the conditional at line 30 should remain.

**Bug 3 — unused role field** (`src/lib/sessionService.ts` `loadRoomContext`):
In the messages loop (around line 1277), the current format is:
```ts
parts.push(`- ${m.agentId}: ${(m.content || '').slice(0, 150)} (${ago})`);
```
Update to include role when it differs from 'agent':
```ts
const label = m.role && m.role !== 'agent' ? `[${m.role}] ${m.agentId}` : m.agentId;
parts.push(`- ${label}: ${(m.content || '').slice(0, 150)} (${ago})`);
```
