    **3. Create `src/components/writing/ChapterEditor.tsx`:**

    TipTap editor wrapper with autosave:

    ```tsx
    import { useEditor, EditorContent } from '@tiptap/react';
    import StarterKit from '@tiptap/starter-kit';
    import Highlight from '@tiptap/extension-highlight';
    import Placeholder from '@tiptap/extension-placeholder';
    import CharacterCount from '@tiptap/extension-character-count';
    import Typography from '@tiptap/extension-typography';
    import Link from '@tiptap/extension-link';
    import EditorToolbar from './EditorToolbar';
    import { useWritingStore } from '../../store/writingStore';
    import { useEffect, useRef, useCallback } from 'react';
    import '../../styles/writing-editor.css';
    ```

    **Critical pitfalls to handle:**
    - C1: TipTap memory leak — call `editor.destroy()` in useEffect cleanup
    - C2: React re-render storm — use `shouldRerenderOnTransaction: false` on useEditor
    - H5: Main process blocking — debounce saves (1.5 second debounce)

    Implementation:
    - `useEditor()` with extensions: StarterKit (includes bold, italic, headings, lists, blockquote, undo/redo, history), Highlight, Placeholder (configure with "Start writing..."), CharacterCount, Typography, Link.configure({ openOnClick: false, HTMLAttributes: { class: 'editor-link' } })
    - Set `shouldRerenderOnTransaction: false` to avoid re-render storm
    - `onUpdate({ editor })` handler: get HTML content, set `setActiveChapterContent(editor.getHTML())`, set `setChapterDirty(true)`, trigger debounced save
    - Debounce implementation: useRef for timer, useCallback for debounced save that calls `saveChapter(content)` after 1500ms of inactivity. Clear timer on each keystroke.
    - When `activeChapterContent` changes from store (opening new chapter), call `editor.commands.setContent(content)` — but only if the content differs from current editor content to avoid cursor reset
    - On unmount: clear debounce timer, save if dirty (flush), destroy editor
    - Word count display: use CharacterCount extension's `editor.storage.characterCount.words()` — show in bottom bar

    Render structure:
    ```tsx
    <div className="writing-editor flex flex-col h-full">
      <EditorToolbar editor={editor} />
      <div className="flex-1 overflow-y-auto">
        <EditorContent editor={editor} />
      </div>
      <div className="border-t border-clawd-border px-4 py-2 text-xs text-clawd-text-dim flex justify-between">
        <span>{wordCount} words</span>
        <span>{chapterDirty ? 'Saving...' : 'Saved'}</span>
      </div>
    </div>
    ```

    **Performance (EDIT-03):** TipTap/ProseMirror handles 10k+ words natively. The key perf concerns are: (a) `shouldRerenderOnTransaction: false` prevents React re-renders on every keystroke, (b) debounced saves prevent IPC spam, (c) no storing full document in React state on every keystroke (only on save).
