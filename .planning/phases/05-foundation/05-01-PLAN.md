<task type="auto">
  <name>Task 2: Create writing-project-service.ts with all IPC handlers</name>
  <files>
    electron/writing-project-service.ts
  </files>
  <action>
    Create `electron/writing-project-service.ts` following the `x-automations-service.ts` pattern (pure functions + registerXxxHandlers export).

    **File storage layout per project:**
    ```
    ~/froggo/writing-projects/{projectId}/
      project.json          # { id, title, type, createdAt, updatedAt }
      chapters/             # Chapter markdown files
        01-{slug}.md        # Chapter content (markdown on disk)
      chapters.json         # Chapter ordering + metadata: [{ id, title, filename, position, createdAt, updatedAt }]
      memory/               # (empty for now, Phase 7)
      versions/             # (empty for now, Phase 9)
    ```

    **Project operations (all async, use fs.promises):**

    1. `listProjects()` — Read WRITING_PROJECTS_DIR, for each subdir read project.json + chapters.json, return array of `{ id, title, type, chapterCount, wordCount, createdAt, updatedAt }`. Word count = sum reading each chapter file and splitting on whitespace. If WRITING_PROJECTS_DIR doesn't exist, create it and return [].

    2. `createProject(title: string, type: 'memoir' | 'novel')` — Generate projectId as `proj-{timestamp}-{random6}`. Create project directory, chapters/ subdir, memory/ subdir, versions/ subdir. Write project.json with id, title, type, timestamps. Write empty chapters.json array. Return the project object.

    3. `getProject(projectId: string)` — Read project.json. Read chapters.json. For each chapter, read the .md file to compute word count. Return `{ ...project, chapters: [...withWordCounts] }`.

    4. `updateProject(projectId: string, updates: { title?: string, type?: string })` — Read project.json, merge updates, update `updatedAt`, write back.

    5. `deleteProject(projectId: string)` — Remove project directory recursively (`fs.promises.rm(dir, { recursive: true, force: true })`).

    **Chapter operations:**

    6. `listChapters(projectId: string)` — Read chapters.json, for each chapter read .md file for word count. Return array with word counts.

    7. `createChapter(projectId: string, title: string)` — Generate chapterId `ch-{timestamp}-{random6}`. Determine position (max position + 1 from chapters.json). Create filename: `{position padded 2 digits}-{slugify(title)}.md`. Write empty .md file. Append to chapters.json. Return chapter metadata.

    8. `readChapter(projectId: string, chapterId: string)` — Find chapter in chapters.json by id. Read .md file. Return `{ ...metadata, content: string, wordCount: number }`.

    9. `saveChapter(projectId: string, chapterId: string, content: string)` — Find chapter in chapters.json. Write content to .md file. Update `updatedAt` in chapters.json. Update project.json `updatedAt`. Return `{ success: true, wordCount }`.

    10. `renameChapter(projectId: string, chapterId: string, newTitle: string)` — Find in chapters.json, update title. Optionally rename file on disk (generate new filename from title, use fs.promises.rename). Update chapters.json entry.

    11. `reorderChapters(projectId: string, chapterIds: string[])` — Receive new order as array of chapter IDs. Update position fields in chapters.json. Optionally rename files to maintain position prefix ordering. Write updated chapters.json.

    12. `deleteChapter(projectId: string, chapterId: string)` — Find in chapters.json, remove .md file, remove from chapters.json.

    **Helper: `slugify(text: string): string`** — lowercase, replace non-alphanumeric with hyphens, collapse multiple hyphens, trim hyphens.

    **Helper: `countWords(content: string): number`** — `content.trim().split(/\s+/).filter(Boolean).length` (return 0 for empty string).

    **IPC registration (export function registerWritingProjectHandlers()):**
    ```typescript
    ipcMain.handle('writing:project:list', async () => listProjects());
    ipcMain.handle('writing:project:create', async (_, title: string, type: string) => createProject(title, type as any));
    ipcMain.handle('writing:project:get', async (_, projectId: string) => getProject(projectId));
    ipcMain.handle('writing:project:update', async (_, projectId: string, updates: any) => updateProject(projectId, updates));
    ipcMain.handle('writing:project:delete', async (_, projectId: string) => deleteProject(projectId));
    ipcMain.handle('writing:chapter:list', async (_, projectId: string) => listChapters(projectId));
    ipcMain.handle('writing:chapter:create', async (_, projectId: string, title: string) => createChapter(projectId, title));
    ipcMain.handle('writing:chapter:read', async (_, projectId: string, chapterId: string) => readChapter(projectId, chapterId));
    ipcMain.handle('writing:chapter:save', async (_, projectId: string, chapterId: string, content: string) => saveChapter(projectId, chapterId, content));
    ipcMain.handle('writing:chapter:rename', async (_, projectId: string, chapterId: string, title: string) => renameChapter(projectId, chapterId, title));
    ipcMain.handle('writing:chapter:reorder', async (_, projectId: string, chapterIds: string[]) => reorderChapters(projectId, chapterIds));
    ipcMain.handle('writing:chapter:delete', async (_, projectId: string, chapterId: string) => deleteChapter(projectId, chapterId));
    ```

    Every handler wraps in try/catch and returns `{ success: boolean, error?: string, ...data }`.

    Import paths from `./paths` (WRITING_PROJECTS_DIR, writingProjectPath, writingChapterPath). Use `import * as fs from 'fs'` with `fs.promises` for all I/O (no sync operations — main process must not block).
  </action>
