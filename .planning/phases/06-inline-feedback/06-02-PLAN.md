**FeedbackPopover.tsx** — Main feedback UI at `src/components/writing/FeedbackPopover.tsx`:

Props: `{ editor: Editor }` where Editor is from `@tiptap/react`.

This component orchestrates the entire feedback flow:

**1. Imports:**
```typescript
import { useState, useRef } from 'react';
import { Editor } from '@tiptap/react';
import { Send, X, Loader2 } from 'lucide-react';
import { gateway } from '../../lib/gateway';
import { useWritingStore } from '../../store/writingStore';
import { useFeedbackStore } from '../../store/feedbackStore';
import AgentPicker from './AgentPicker';
import FeedbackAlternative from './FeedbackAlternative';
```

**2. State management:**
- Read from feedbackStore: `selectedAgent`, `instructions`, `streaming`, `streamContent`, `alternatives`, `error`, `savedSelection` + setters + `reset`
- Read from writingStore: `activeProjectId`, `activeChapterId`, `activeChapterContent`, `activeProject`

**3. Helper: `getSelectedText()`**
```typescript
const getSelectedText = () => {
  const { from, to } = editor.state.selection;
  return editor.state.doc.textBetween(from, to, ' ');
};
```

**4. Helper: `buildPrompt(selectedText, instructions, chapterContent, outline)`**
Build the AI prompt following the pattern from research (Pattern 5):
- Include `## Task` with selected text and user instructions
- Include `### Chapter Context` — truncate chapterContent to ~2000 words around the selection position (use Pitfall 6 mitigation). Get the selection position from `editor.state.selection.from`, find the approximate character offset, then slice ~8000 chars before/after.
- Include `### Project Outline` — build from `activeProject.chapters` (chapter titles in order). If no chapters, say "(no outline available)".
- Include `## Response Format` requesting exactly 3 alternatives formatted as `### Alternative 1/2/3`
- Agent-specific preamble: if writer, mention "style, pacing, narrative"; if researcher, "accuracy, fact-checking, clarity"; if jess, "emotional impact, sensitivity, memoir-specific tone"

**5. Helper: `parseAlternatives(response: string): string[]`**
```typescript
const parts = response.split(/###\s*Alternative\s*\d+\s*/i);
return parts.slice(1).map(p => p.trim()).filter(Boolean).slice(0, 3);
```

**6. Handler: `handleSend()`**
- Get selected text; if empty or instructions empty, return
- Save the current selection range: `setSavedSelection({ from, to })` from `editor.state.selection`
- Set streaming true, clear streamContent/alternatives/error
- Build project-scoped session key: `agent:${selectedAgent}:writing:${activeProjectId}`
- Build prompt with `buildPrompt()`
- Call `gateway.sendChatWithCallbacks(prompt, sessionKey, { onDelta, onEnd, onError })`
  - `onDelta`: accumulate into ref, call `setStreamContent(accumulated)`
  - `onEnd`: parse alternatives from accumulated text with `parseAlternatives()`, call `setAlternatives(parsed)`, set streaming false
  - `onError`: set error, set streaming false
- Log the interaction via IPC: `(window as any).clawdbot?.writing?.feedback?.log(activeProjectId, { chapterId, agentId, selectedText, instructions, alternatives, accepted: null, selectionRange })`

**7. Handler: `handleAccept(alternativeText: string)`**
- Get savedSelection from store
- If savedSelection exists, use it; otherwise fall back to current editor selection
- Call `editor.chain().focus().insertContentAt({ from, to }, alternativeText, { updateSelection: true }).run()`
- After replacement, collapse selection to prevent BubbleMenu flicker: `editor.commands.setTextSelection(from + alternativeText.length)` (Pitfall 1 mitigation)
- Log the accept via IPC (update entry with `accepted: alternativeText`)
- Call `reset()` on feedbackStore

**8. Handler: `handleDismiss()`**
- Call `reset()` on feedbackStore
- Optionally collapse selection: `editor.commands.setTextSelection(editor.state.selection.to)`

**9. Render:**
```
<div className="bg-clawd-surface border border-clawd-border rounded-lg shadow-lg p-3 min-w-[320px] max-w-[480px]"
     onMouseDown={(e) => e.preventDefault()} // Prevent editor blur (Pitfall 7 backup)
>
  {/* Agent picker row */}
  <AgentPicker selected={selectedAgent} onSelect={setSelectedAgent} disabled={streaming} />

  {/* Instruction input + send button */}
  <div className="flex gap-2 mt-2">
    <input
      type="text"
      value={instructions}
      onChange={(e) => setInstructions(e.target.value)}
      onKeyDown={(e) => { if (e.key === 'Enter' && !e.shiftKey) { e.preventDefault(); handleSend(); } }}
      placeholder="How should this be rewritten?"
      className="flex-1 bg-clawd-bg border border-clawd-border rounded px-2 py-1.5 text-sm text-clawd-text placeholder:text-clawd-text-dim focus:outline-none focus:border-clawd-accent"
      disabled={streaming}
      autoFocus
    />
    <button onClick={handleSend} disabled={streaming || !instructions.trim()} className="...">
      {streaming ? <Loader2 className="w-4 h-4 animate-spin" /> : <Send className="w-4 h-4" />}
    </button>
  </div>

  {/* Streaming content (raw text while streaming) */}
  {streaming && streamContent && (
    <div className="mt-2 text-sm text-clawd-text-dim max-h-[200px] overflow-y-auto">
      <pre className="whitespace-pre-wrap font-sans">{streamContent}</pre>
    </div>
  )}

  {/* Parsed alternatives (after streaming completes) */}
  {!streaming && alternatives.length > 0 && (
    <div className="mt-2 space-y-2">
      {alternatives.map((alt, i) => (
        <FeedbackAlternative key={i} index={i} text={alt} onAccept={handleAccept} />
      ))}
      <button onClick={handleDismiss} className="text-xs text-clawd-text-dim hover:text-clawd-text">
        Dismiss
      </button>
    </div>
  )}

  {/* Error state */}
  {error && (
    <div className="mt-2 text-sm text-red-400">{error}</div>
  )}
</div>
```

IMPORTANT: The `onMouseDown={(e) => e.preventDefault()}` on the root div is critical. This prevents the editor from losing focus when the user clicks inside the popover (Pitfall 7). BubbleMenu has built-in preventHide for its own element, but this extra guard prevents focus loss on the input and buttons inside.

IMPORTANT: Use a `useRef` for accumulating stream content (not just state), because `onDelta` closures can capture stale state. The ref accumulates, then state is set from the ref value.

**2. Add BubbleMenu after `<EditorContent editor={editor} />`** inside the `flex-1 overflow-y-auto` div:

```typescript
<div className="flex-1 overflow-y-auto">
  <EditorContent editor={editor} />
  {editor && (
    <BubbleMenu
      editor={editor}
      shouldShow={({ editor: ed, from, to }) => {
        // Only show for non-empty text selections
        if (from === to) return false;
        return !ed.state.selection.empty;
      }}
      updateDelay={0}
      options={{
        placement: 'top',
        offset: { mainAxis: 8 },
        flip: true,
        shift: { padding: 8 },
      }}
    >
      <FeedbackPopover editor={editor} />
    </BubbleMenu>
  )}
</div>
```

**Key notes:**
- `updateDelay={0}` — instant popup, not the sluggish 250ms default (Pitfall 4)
- `shouldShow` checks both `from === to` (collapsed cursor) and `selection.empty` (node selections)
- `placement: 'top'` with flip — shows above selection, flips below if near top edge
- `shift: { padding: 8 }` — keeps popover within viewport with 8px margin
- The BubbleMenu MUST be inside the same scroll container as EditorContent so it tracks scroll position correctly

**Do NOT change** any of the existing editor setup, autosave, word count, or toolbar code. This is a purely additive change.
