<task type="auto">
  <name>Task 1: Post-conversation memory extraction</name>
  <files>src/lib/sessionService.ts, app/api/sessions/chat/route.ts</files>
  <action>
  After every N messages (configurable, default 5) or on session close, extract learnings:

  1. Add function `extractAndSaveMemory(sessionKey: string, agentId: string)`:
     - Load last 10 messages from the session
     - Send to Gemini Flash Lite with prompt:
       "Extract key learnings, decisions, user preferences, and patterns from this conversation.
        Return as bullet points. Only include NEW information worth remembering.
        Skip greetings, small talk, and already-known facts.
        Format: one bullet per learning, concise."
     - Parse response into bullet points
     - Append to `~/mission-control/memory/agents/{agentId}/learnings.md` with date header
     - If file > 5000 chars, summarize older entries (keep last 20 entries verbatim, summarize rest)

  2. In /api/sessions/chat, after every 5th message in a session:
     - Fire-and-forget call to extractAndSaveMemory()
     - Don't block the response

  3. Track extraction state in sessions table: `last_memory_extract_at` column
     - Skip if extracted less than 5 minutes ago (prevent spam)

  This means: every 5 messages, the agent's memory grows. Next session, it remembers.
  </action>
  <verify>npx tsc --noEmit. After 5 messages in a session, learnings.md updated in agent memory dir.</verify>
  <done>Memory extraction runs automatically. Agent learnings persist to vault. Old learnings summarized to prevent bloat.</done>
</task>
