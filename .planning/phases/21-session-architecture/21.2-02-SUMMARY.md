# Phase 21.2 Plan 02: Memory Write-Back Summary

**Agents now save learnings after conversations — memory accumulates automatically. Every 5th message triggers extraction. Old learnings are summarized to prevent bloat. Memory protocol appears in every system prompt.**

## Performance

- **Duration:** <1 min (implementation pre-existed from commit d0b8408f)
- **Completed:** 2026-03-17
- **Tasks:** 2
- **Files modified:** 2

## Accomplishments

### Task 1: Post-conversation memory extraction

`extractAndSaveMemory(sessionKey, agentId)` is implemented in `src/lib/sessionService.ts`:

- Checks 5-minute cooldown via `lastMemoryExtractAt` column in sessions table
- Loads last 10 messages from the session
- Sends to Gemini Flash Lite (fallback: gemini-2.5-flash) with extraction prompt: extract learnings, decisions, preferences, patterns — bullet points only, skip greetings/small talk, return "NONE" if nothing worth saving
- Parses bullet points (`- ` or `* ` prefixed lines)
- Appends to `~/mission-control/memory/agents/{agentId}/learnings.md` with `### YYYY-MM-DD — session {id}` date header
- When file exceeds 5000 chars: `summarizeOlderLearnings()` keeps last 20 entries verbatim, summarizes older sections via Gemini into a compact bullet list
- Updates `lastMemoryExtractAt` timestamp after every run (even when NONE returned)

In `app/api/sessions/chat/route.ts`:
- After every 5th message (`session.messageCount % MEMORY_EXTRACT_EVERY_N === 0`), fires `extractAndSaveMemory()` as a promise with `.catch()` — never blocks response

### Task 2: Memory protocol in system prompt

`buildMemoryProtocol(meta: MemoryMetadata)` injects into every system prompt via `buildSessionContext()`:

- "You have persistent memory. Your memory files are loaded above."
- Explicit list of what to remember: user preferences, project decisions, patterns, context
- "Your memory is automatically extracted every 5 messages"
- "Reference your memory files when relevant — don't re-ask questions you already know the answer to"
- "If your memory contains outdated information, note it in your response"
- Dynamic metadata section:
  - Empty state: "You have no memory files yet. Your learnings from this conversation will be saved automatically."
  - Has files: "Your memory files were last updated: YYYY-MM-DD" + "You have N memory files totaling M characters."

## How Memory Write-Back Works

**Trigger**: Every 5th message in any session (messageCount % 5 === 0), after the agent response is persisted.

**What's extracted**: Key learnings, decisions, user preferences, patterns from the last 10 messages. Gemini determines what's NEW and worth remembering — greetings and small talk are filtered out.

**Where it's saved**: `~/mission-control/memory/agents/{agentId}/learnings.md`
- Appended with date headers so history is navigable
- Auto-condensed via Gemini when file exceeds 5000 chars

**Agent awareness**: Every system prompt contains the memory protocol explaining that memory is live and growing. Agents know to reference past learnings instead of re-asking known questions.

**Non-blocking**: Extraction runs async in the background. Chat responses are never delayed waiting for memory writes.
