# Phase 21.4 Plan 01: Surface-Specific Context — Summary

**Every chat surface now injects rich, relevant context into the agent session. Task agents know their acceptance criteria and subtasks. Social agents have live X pipeline data. Room agents know who else is in the room. 1-1 chat agents know their own status and current work.**

## Performance

- **Duration:** <1 min (implementation pre-existed from commit 2e64491f)
- **Completed:** 2026-03-17
- **Tasks:** 2
- **Files modified:** 2

## Accomplishments

### Task 1: Task surface context loader

`loadTaskContext(taskId, agentId)` in `src/lib/sessionService.ts`:

- Queries `tasks` table: title, description, status, priority, planningNotes, assignedTo, project, project_id, tags, dueDate
- Queries `subtasks` table: all subtasks with `- [x]` / `- [ ]` completion formatting
- Queries `task_activity` table: last 10 entries in reverse-chronological (oldest first), capped at 120 chars each
- Queries `task_attachments`: file names for attachment list
- If `project_id` present: resolves project name + goal from `projects` table
- Formatted under `--- TASK CONTEXT ---` header, truncated to 2000 tokens

Integrated into `buildSessionContext()`: when `surface === 'task'`, calls `loadTaskContext(contextId, agentId)`.

### Task 2: Social + room + main chat context loaders

`loadSocialContext(tab)` — moved from `generate-reply/route.ts` into sessionService:
- `pipeline` / `configure`: pipeline post status counts, last 5 recent posts
- `engage`: mention reply status breakdown, last 5 pending mentions
- `intelligence`: tracked competitor handles, latest competitor report summary
- `measure`: total mentions + replied count
- `configure`: active automations with trigger type and run counts
- All tabs: brand knowledge from `knowledge_base` (top 3 articles)
- `generate-reply` route refactored to `import { loadSocialContext } from '@/lib/sessionService'` (removed ~100 lines of duplicated `fetchTabData` function)

`loadRoomContext(roomId)`:
- Queries `chat_rooms`: name, topic, description, agents list
- Loads last 5 messages from `chat_room_messages` for shared conversation context
- Formats agent list from JSON `agents` column

`loadMainChatContext(agentId)`:
- Loads agent status + `currentTaskId` from `agents` table
- Resolves current task title/status/priority from `tasks` if `currentTaskId` set
- Loads last 5 activity entries from `task_activity WHERE agentId`

All 4 loaders wired into `buildSessionContext()`:
```ts
surface === 'task'   → loadTaskContext(contextId, agentId)
surface === 'social' → loadSocialContext(metadata.tab)
surface === 'room'   → loadRoomContext(contextId)
surface === 'chat'   → loadMainChatContext(agentId)
```

## What Each Surface Gets

| Surface | Context injected |
|---------|-----------------|
| Task chat | Task title, description, planning notes/acceptance criteria, subtask checklist, last 10 activity entries, file attachments, project name + goal |
| Social chat | Tab-specific live stats (pipeline counts, pending mentions, competitors, engagement), brand knowledge from KB |
| Room chat | Room name, topic, description, participating agents list, last 5 room messages |
| 1-1 chat | Agent status (busy/idle), current task title + status, last 5 activity entries |
