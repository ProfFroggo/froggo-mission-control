## Accomplishments

- `loadAgentMemory(agentId)` reads all `.md` files from `~/mission-control/memory/agents/{agentId}/`, sorted newest first, with 2000 token budget. Creates the directory if it doesn't exist so new agents get a memory home automatically. Also loads `~/mission-control/memory/shared/` at 500 token budget.
- `loadKnowledgeBase(agentId, surface, metadata)` queries the `knowledge_base` table for articles with `scope='agents'` or `scope='all'`. Scores articles by tag relevance (social-manager gets +10 for brand/voice/strategy/x-twitter tags, task sessions get +5 for project tag match). Voice & Style Guide pinned to score 1000 for social-manager so it is always included. Top 5 articles at 1500 token budget.
- Both functions integrated into `buildSessionContext()` — `memoryContext` and `knowledgeContext` fields populated on every session.
- Context order in system prompt: identity (SOUL.md) → surface context → memory → memory protocol → knowledge base.

## Agents Can Query KB Mid-Conversation

Yes. The KB context is injected into the system prompt on every `buildSessionContext()` call, which means agents always have KB articles available in their context window during the full conversation. For social-manager, the Voice & Style Guide is guaranteed present. For task sessions, articles matching the task's project tags are prioritized. Agents can reference KB content in their responses without any tool call — it's already in context.
