# Phase 21.3 Plan 01: Context Compaction — Summary

**Long conversations no longer degrade. When a session exceeds 25 messages, Gemini summarizes older messages into a compact context that persists in the DB. Agents see both the summary and the last 8 messages verbatim on every new message.**

## Performance

- **Duration:** <1 min (implementation pre-existed from commit c4d765a6)
- **Completed:** 2026-03-17
- **Tasks:** 2
- **Files modified:** 3

## Accomplishments

### Task 1: Context compaction engine

`compactConversation(sessionKey)` in `src/lib/sessionService.ts`:

- Loads ALL messages for the session, returns null if < 20 messages
- Splits: keeps last `COMPACTION_RECENT_KEEP` (8) messages verbatim, summarizes the rest
- Sends older messages to Gemini Flash Lite (fallback: gemini-2.5-flash) with compaction prompt preserving decisions, preferences, files, action items, and important context
- Saves summary to `sessions.compact_summary` with timestamp in `sessions.last_compact_at`
- Returns the summary string, or null on failure

`loadConversationHistory(sessionKey)`:
- Checks `sessions.compact_summary` first
- If present: returns `"[Conversation summary from earlier]\n{summary}\n\n[Recent messages]\n{recent 8 messages}"`
- If not: returns last 20 messages as before
- Total budget: 8000 tokens (`BUDGET_HISTORY`)

`buildSessionContext()` trigger:
- Checks `session.messageCount > 25` (COMPACTION_THRESHOLD)
- Checks cooldown: `last_compact_at` must be > 10 minutes ago (COMPACTION_COOLDOWN_MS)
- Fires `compactConversation()` as fire-and-forget — stale summary used until new one is ready

DB columns added (via `columnMigrations` in database.ts):
- `ALTER TABLE sessions ADD COLUMN compact_summary TEXT`
- `ALTER TABLE sessions ADD COLUMN last_compact_at INTEGER`

### Task 2: Session state tracking + cleanup

`getSessionStats(sessionKey)` returns `SessionStats`:
- `messageCount`, `age` (ms since creation), `compacted` (boolean), `lastActivity`, `tokenEstimate`
- `memoryFileCount` (from fs scan of agent memory dir)
- `kbArticleCount` (from knowledge_base table)

`cleanupStaleSessions()`:
- Finds sessions: `createdAt < 30 days ago AND lastActivity < 7 days ago`
- Deletes messages then sessions for each stale key (transaction-safe)
- Also cleans orphaned messages where sessionKey is missing from sessions table
- Returns `{ deleted: number, orphanedMessages: number }`

Cron daemon (`tools/cron-daemon.js`):
- Calls `cleanupStaleSessions()` on startup and every 24 hours
- Implemented directly in cron-daemon (not importing from sessionService — cron uses its own DB handle)

DB indexes added:
- `CREATE INDEX IF NOT EXISTS idx_sessions_agentId ON sessions(agentId)`
- `CREATE INDEX IF NOT EXISTS idx_sessions_lastActivity ON sessions(lastActivity)`

Additional session management functions:
- `resetSession(sessionKey)` — clears messages, compact_summary, last_compact_at
- `deleteSession(sessionKey)` — removes session and all messages
- `exportSessionAsMarkdown(sessionKey)` — returns full conversation as markdown
- `listAllSessions()` — all sessions with stats (for session management UI in Phase 21.5)

## How Compaction Works

**Trigger**: `buildSessionContext()` is called before every agent response. If the session has >25 messages and the last compaction was >10 minutes ago, `compactConversation()` is fired async.

**What happens during compaction**:
1. Load all messages for session in chronological order
2. Take messages from start to `length - 8` (the "older" slice)
3. Format them as `User: ...` / `Agent: ...` lines (each capped at 400 chars)
4. Send to Gemini with prompt: preserve decisions, preferences, files mentioned, action items, important context. Keep under 500 words. Use bullet points.
5. Save Gemini's response to `sessions.compact_summary`
6. Update `sessions.last_compact_at = now`

**What the agent sees** (via `loadConversationHistory()`):
```
[Conversation summary from earlier]
• User prefers concise responses
• Agreed to use TypeScript strict mode
• Working in src/lib/sessionService.ts
• Next: add streaming support to invokeAgent()
...

[Recent messages]
User: What's the token budget for history?
Agent: 8000 tokens — see BUDGET_HISTORY const...
...
(last 8 messages verbatim)
```

**Non-blocking**: The current request uses the existing (possibly stale) `compact_summary`. New summary is available on the NEXT message, ~1-2 seconds later (Gemini Flash Lite is very fast).
