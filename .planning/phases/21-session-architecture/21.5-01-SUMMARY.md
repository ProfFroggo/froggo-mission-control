# Phase 21.5 Plan 01: Session UI — Summary

**Users can see exactly what's in their agent's context at a glance. Every chat surface shows memory files loaded, KB articles, compaction status, and a colour-coded context usage bar. Sessions can be reset, exported, and deleted from settings.**

- **Duration:** <1 min (implementation pre-existed from commits in prior session)
- **Completed:** 2026-03-17
- **Tasks:** 2
- **Files modified:** 7

## What Was Already Built

All implementation was present before this plan ran, committed during the same session that built Phase 21.1–21.4. No new code was required.

## Task 1: Session stats endpoint + header indicators

### `/api/sessions/stats/route.ts`

- `GET /api/sessions/stats?key=` — returns `{ messageCount, age, compacted, lastActivity, tokenEstimate, memoryFileCount, kbArticleCount }` via `getSessionStats()`
- `GET /api/sessions/stats` (no key) — returns all sessions list + summary stats for the settings panel
- `POST /api/sessions/stats` `{ action: 'reset', key }` — clears messages + compact_summary
- `POST /api/sessions/stats` `{ action: 'export', key }` — returns markdown string for download
- `DELETE /api/sessions/stats?key=` — deletes session + all messages

### `SessionStatsBar.tsx`

Reusable component mounted in all chat surfaces:

- `{N} msgs` — message count
- `{N} memory files` / `No memory` (dimmed) — agent memory files on disk
- `{N} KB articles` / `No KB` (dimmed) — knowledge base articles in scope
- `Compacted` badge when `compact_summary` is set
- Context usage progress bar (green ≤50%, warning ≤80%, error >80%) against 32k token baseline
- "New session" reset button (optional, via `showReset` prop)

Wired into:
- `ChatPanel` — `showReset={false}` (reset handled by full panel controls)
- `TaskChatTab` — `onReset={() => setMessages([])}` clears local state
- `AgentChatModal` — `onReset={() => setMessages([])}` clears local state

### `XAgentChatPane.tsx`

Social chat panel implements stats inline (predates SessionStatsBar):
- Fetches on mount and after every message send
- Same indicators: msgs / memory / KB / compacted / context bar
- "New session" button in header (RotateCcw icon)

## Task 2: Session management in settings

### `EnhancedSettingsPanel.tsx` — Sessions tab

`SessionsManagementSection` component:

**Summary stats row (4 cards):**
- Total sessions
- Total messages
- Oldest session (relative age)
- Most active agent

**Session list grouped by surface** (Chat, Social, Task, Room, Cron, Library, Other):
- Agent name, session key, message count, last active, created
- `compacted` badge when compacted
- Export button (Download icon) — downloads `session-{key}.md` via blob URL
- Delete button (Trash2 icon) — confirms then calls `DELETE /api/sessions/stats?key=`

Sessions tab registered as `'sessions'` in the tab type union and rendered in the tab sidebar under `Terminal` icon.
