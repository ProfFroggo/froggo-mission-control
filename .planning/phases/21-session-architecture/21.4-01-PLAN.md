<task type="auto">
  <name>Task 1: Task surface context loader</name>
  <files>src/lib/sessionService.ts</files>
  <action>
  Add function `loadTaskContext(taskId: string, agentId: string): string`:

  1. Query tasks table for the task: title, description, status, priority, planningNotes, assignedTo, project, project_id, tags, dueDate
  2. Query subtasks table: all subtasks with completion status
  3. Query task_activity table: last 10 activity entries (who did what, when)
  4. Query task_attachments: list of attached files with paths
  5. If task has project_id: load project name and goal from projects table

  Format:
  ```
  --- TASK CONTEXT ---
  You are working on this task. Use this context to inform your responses.

  **Task**: {title}
  **Status**: {status} | **Priority**: {priority} | **Assigned to**: {assignedTo}
  **Project**: {project name} — {project goal}
  **Due**: {dueDate or 'No deadline'}

  **Description**:
  {description}

  **Planning Notes / Acceptance Criteria**:
  {planningNotes}

  **Subtasks**:
  - [x] {completed subtask}
  - [ ] {pending subtask}

  **Recent Activity**:
  - {agent} {action}: {message} ({relative time})

  **Attachments**: {file list}
  ```

  Budget: 2000 tokens for task context.

  2. Integrate into `buildSessionContext()`: when surface === 'task', call loadTaskContext(contextId, agentId) and set as surfaceContext.
  </action>
  <verify>npx tsc --noEmit. Task context loads correctly for a real task ID. Includes planning notes and subtasks.</verify>
  <done>Task chat sessions have full task context: description, planning notes, subtasks, activity, attachments.</done>
</task>

<task type="auto">
  <name>Task 2: Social + room + main chat context loaders</name>
  <files>src/lib/sessionService.ts</files>
  <action>
  Move the existing `fetchTabData()` logic from generate-reply into sessionService as `loadSocialContext(tab: string)`.

  Then add:

  `loadRoomContext(roomId: string)`:
  - Query chat_rooms table for room metadata (name, agents, created)
  - Load last 5 messages from chat_room_messages for shared context
  - List participating agents
  - Format as "Room: {name}, Agents: {list}, Recent: {messages}"

  `loadMainChatContext(agentId: string)`:
  - Load agent's current task (from agents table: currentTaskId)
  - Load agent's recent activity (from task_activity WHERE agentId, last 5)
  - Load agent's status (busy/idle)
  - Format as "Agent status: {status}, Current task: {title or 'none'}, Recent: {activity}"

  Integrate all into `buildSessionContext()`:
  - surface === 'social' → loadSocialContext(contextId)
  - surface === 'room' → loadRoomContext(contextId)
  - surface === 'chat' → loadMainChatContext(agentId)
  - surface === 'task' → loadTaskContext(contextId, agentId)

  Remove the fetchTabData function from generate-reply (it's now in sessionService).
  </action>
  <verify>npx tsc --noEmit. Each surface type returns appropriate context. Social loads live data, task loads task details, room loads room state.</verify>
  <done>All 4 surface types have dedicated context loaders. Context is rich and relevant per surface.</done>
