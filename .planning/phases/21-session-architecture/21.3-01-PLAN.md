<task type="auto">
  <name>Task 1: Build context compaction engine</name>
  <files>src/lib/sessionService.ts</files>
  <action>
  Add compaction to the session service:

  1. Add `sessions` table column: `compact_summary TEXT` (via ALTER TABLE IF NOT EXISTS in database.ts)

  2. Add function `compactConversation(sessionKey: string)`:
     - Load all messages for the session
     - If < 20 messages, return null (no compaction needed)
     - Split: keep last 8 messages verbatim (recent context), summarize the rest
     - Send older messages to Gemini Flash Lite with prompt:
       "Summarize this conversation, preserving:
        - Key decisions made
        - User preferences expressed
        - Files or resources mentioned
        - Action items agreed upon
        - Important context that would be lost
        Keep under 500 words. Use bullet points."
     - Save summary to sessions.compact_summary
     - Return summary

  3. Update `loadConversationHistory()`:
     - Check sessions.compact_summary first
     - If exists: return "[Conversation summary from earlier]\n{summary}\n\n[Recent messages]\n{last 8 messages}"
     - If not: return last 20 messages as before
     - Token budget for history: 8000 tokens (summary ~500 + recent ~7500)

  4. Trigger compaction in `buildSessionContext()`:
     - Count total messages for session
     - If > 25 and no recent compaction (last_compact_at > 10 min ago): run compactConversation()
     - Don't block — run async, use stale summary until new one ready
     - Store `last_compact_at` timestamp in sessions table

  5. Add `last_compact_at INTEGER` column to sessions table (database.ts)
  </action>
  <verify>npx tsc --noEmit. After 25+ messages, compact_summary populated. History includes summary + recent messages.</verify>
  <done>Long conversations are compacted. Summary preserves key context. Recent messages stay verbatim.</done>
</task>
