<task type="auto">
  <name>Task 1: Build sessionService.ts — the core session engine</name>
  <files>src/lib/sessionService.ts</files>
  <action>
  Create `src/lib/sessionService.ts` with:

  ```typescript
  interface SessionConfig {
    sessionKey: string;       // e.g. "social:social-manager:pipeline"
    agentId: string;          // e.g. "social-manager"
    surface: 'chat' | 'task' | 'social' | 'room' | 'library';
    contextId?: string;       // tab name, task ID, room ID
    metadata?: Record<string, any>; // surface-specific data (task planningNotes, tab name, etc.)
  }

  interface SessionContext {
    systemPrompt: string;     // SOUL.md + identity + constraints
    conversationHistory: string; // recent messages (compacted if needed)
    memoryContext: string;    // agent memory files
    knowledgeContext: string; // relevant KB articles
    surfaceContext: string;   // surface-specific data (tab data, task context, etc.)
    totalTokenEstimate: number;
  }
  ```

  Functions:
  - `createOrGetSession(config: SessionConfig)`: upsert into sessions table, return session record
  - `loadAgentIdentity(agentId: string)`: read SOUL.md from `catalog/agents/{id}/soul.md` or `.claude/agents/{id}.md`, return first 3000 chars
  - `loadConversationHistory(sessionKey: string, maxMessages: number)`: read from messages table, return formatted history string with token estimate
  - `saveMessage(sessionKey: string, role: 'user' | 'assistant', content: string)`: insert into messages table with timestamp
  - `buildSessionContext(config: SessionConfig)`: orchestrates all loaders, returns SessionContext with assembled prompt
  - `invokeAgent(config: SessionConfig, message: string)`: builds context + calls Claude CLI with --print, returns response text

  Key implementation details:
  - SOUL.md loading: try catalog/agents/{id}/soul.md first, then .claude/agents/{id}.md, then DB agents table description field
  - History: load last 20 messages for the session key, format as "User: ...\nAgent: ..." pairs
  - Token estimation: rough count (chars / 4) for budget tracking
  - Context budget: 8000 tokens for history, 3000 for SOUL.md, 2000 for memory, 1500 for KB, 2000 for surface data = ~16,500 total (well within Haiku's 200K)
  - Claude invocation: spawnSync with --print, --model from ENV.MODEL_TRIVIAL, --system-prompt with full assembled context
  - Clean env vars (strip CLAUDECODE, CLAUDE_CODE_ENTRYPOINT, CLAUDE_CODE_SESSION_ID)
  </action>
  <verify>npx tsc --noEmit passes. Module exports all functions. Can be imported from API routes.</verify>
  <done>sessionService.ts exists with all functions. Types are clean. No external dependencies added.</done>
</task>

<task type="auto">
  <name>Task 2: Build POST /api/sessions/chat — unified chat endpoint</name>
  <files>app/api/sessions/chat/route.ts</files>
  <action>
  Create `app/api/sessions/chat/route.ts`:

  POST body:
  ```json
  {
    "message": "user's message text",
    "sessionKey": "social:social-manager:pipeline",
    "agentId": "social-manager",
    "surface": "social",
    "contextId": "pipeline",
    "metadata": { "tab": "pipeline" }
  }
  ```

  Handler:
  1. Validate required fields (message, sessionKey, agentId, surface)
  2. Call `saveMessage(sessionKey, 'user', message)` to persist user message
  3. Call `buildSessionContext({ sessionKey, agentId, surface, contextId, metadata })`
  4. Call `invokeAgent()` with the assembled context + message
  5. Call `saveMessage(sessionKey, 'assistant', response)` to persist agent response
  6. Return `{ success: true, reply: response, sessionKey, tokenEstimate }`

  Error handling:
  - If Claude CLI fails: return 500 with error detail
  - If agent not found: return 404
  - Timeout: 45 seconds (longer than current 30s to accommodate larger context)

  This replaces `/api/chat/generate-reply` for all new integrations.
  The old endpoint stays for backward compat but new code should use this.
  </action>
  <verify>npx tsc --noEmit passes. curl -X POST /api/sessions/chat with test data returns a response. Messages persisted in DB.</verify>
  <done>Endpoint works, persists messages, returns agent response. Session created in sessions table.</done>
