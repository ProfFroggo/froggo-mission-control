Key implementation details:
- `Group orientation="horizontal"` wrapping 3 `Panel` components with 2 `Separator` components
- `defaultLayout` reads from localStorage key `'writing-layout'`, falls back to `[15, 30, 55]`
- `onLayoutChanged` callback saves layout array to localStorage as JSON
- Left panel (chapters): `id="chapters"`, `minSize={10}`, `maxSize={25}`, `collapsible`, `collapsedSize={0}`
- Center panel (chat placeholder): `id="chat"`, `minSize={15}`, `maxSize={50}`, `collapsible`, `collapsedSize={0}`. Render a placeholder div: `<div className="flex items-center justify-center h-full text-clawd-text-dim text-sm">Chat pane (coming soon)</div>` — Plan 03 will replace this with ChatPane.
- Right panel (editor): `id="editor"`, `minSize={25}`. Contains editor + context/version overlays.
- Use `useRef<ImperativePanelHandle>` for `chaptersPanelRef` and `chatPanelRef` to provide collapse/expand toggle buttons.
- Move context/version panels INSIDE the editor panel as absolutely-positioned overlays (right-aligned), keeping the existing toggle button logic.
- Add collapse/expand toggle buttons in a toolbar bar between the separator and the panel content. Use `PanelLeftClose`/`PanelLeftOpen` icons. Toggle by calling `panelRef.current?.collapse()` / `panelRef.current?.expand()`.
- Collapse state tracking: use `onCollapse` and `onExpand` callbacks on each collapsible Panel to track `isChaptersCollapsed` and `isChatCollapsed` state.
