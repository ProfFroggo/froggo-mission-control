<objective>
Living knowledge base that automatically discovers insights from completed tasks and flags stale articles.

Purpose: Knowledge accumulates without manual effort — agents discover and document learnings.
Output: Daily knowledge review API, freshness scoring, auto-link related articles.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Create daily knowledge review API + cron job config</name>
  <files>app/api/knowledge/review/route.ts, tools/cron-daemon.js (schedule entry)</files>
  <action>
  Create POST /api/knowledge/review endpoint that:
  1. Queries tasks completed in last 24h (status='done', completedAt > 24h ago)
  2. For each task: check if a knowledge article already references this task (by taskId in tags)
  3. For new tasks: extract title, planningNotes, activity log, and attachments
  4. Generate a knowledge article via Gemini (POST to /api/knowledge/generate with structured prompt)
  5. Save to knowledge_base with category='auto-discovered', tags including task ID
  6. Return count of articles created

  Add a cron schedule.json entry for daily 8am UTC run targeting this endpoint.
  </action>
  <verify>npx tsc --noEmit passes</verify>
  <done>Daily knowledge review creates articles from completed tasks.</done>
</task>

<task type="auto">
  <name>Task 2: Knowledge freshness scoring + auto-linking</name>
  <files>app/api/knowledge/[id]/route.ts</files>
  <action>
  1. Add freshnessScore computed field to knowledge article GET responses:
     - Score = 100 - (daysSinceUpdate * 2). Min 0, max 100.
     - Articles >30 days old get score 40 (stale warning)
     - Articles >90 days old get score 0 (needs review)
  2. Add relatedArticles field: query knowledge_base for articles with overlapping tags (top 3)
  3. These are read-only computed fields, not stored in DB
  </action>
  <verify>npx tsc --noEmit passes</verify>
  <done>Knowledge articles have freshness scores and related article links.</done>
</task>

</tasks>
