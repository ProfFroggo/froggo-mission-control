Create `src/store/memoryStore.ts` following the exact pattern of `writingStore.ts` and `feedbackStore.ts`:

**Interfaces** (renderer-side, no timestamps — those are managed by the IPC layer):
- `CharacterProfile`: id, name, relationship, description, traits (string[])
- `TimelineEvent`: id, date, description, chapterRefs (string[]), position
- `VerifiedFact`: id, claim, source, status ('unverified' | 'verified' | 'disputed')

**State interface** `MemoryState`:
- `characters: CharacterProfile[]`
- `timeline: TimelineEvent[]`
- `facts: VerifiedFact[]`
- `loading: boolean`
- `activeTab: 'characters' | 'timeline' | 'facts'`
- `editingId: string | null` (which entity is currently being edited in the form)

**Actions:**
- `loadMemory(projectId)`: call bridge().characters.list, bridge().timeline.list, bridge().facts.list in parallel (Promise.all), set all three arrays. Set loading during operation.
- `addCharacter(projectId, data)`: call bridge().characters.create, on success reload characters list
- `updateCharacter(projectId, id, data)`: call bridge().characters.update, on success reload characters list
- `deleteCharacter(projectId, id)`: call bridge().characters.delete, on success reload characters list
- Same 3 actions for timeline (addTimelineEvent, updateTimelineEvent, deleteTimelineEvent)
- Same 3 actions for facts (addFact, updateFact, deleteFact)
- `setActiveTab(tab)`: set activeTab
- `setEditingId(id)`: set editingId
- `clearMemory()`: reset characters, timeline, facts to empty arrays, editingId to null

**Bridge accessor:** `const bridge = () => (window as any).clawdbot?.writing?.memory;`

For reload after mutation: call `bridge().{type}.list(projectId)` and set the corresponding state array. Do NOT re-read all three — only the one that changed.

Export as `useMemoryStore`.
