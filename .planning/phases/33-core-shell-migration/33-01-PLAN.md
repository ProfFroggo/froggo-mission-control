**Radix IconButton API:**
```tsx
import { IconButton } from '@radix-ui/themes';

// Ghost (default inactive state):
<IconButton variant="ghost" size="3" color="gray" aria-label="Settings" title="Settings">
  <Settings size={16} />
</IconButton>

// Solid (active/selected state):
<IconButton variant="solid" size="3" color="grass" aria-label="Settings">
  <Settings size={16} />
</IconButton>
```

Sizes: size="1"=24px, size="2"=32px, size="3"=40px, size="4"=48px
Our current buttons are size-11 (44px) — size="3" (40px) is close enough.

**Critical:** Keep className="no-drag" on each IconButton — this class controls drag behavior in the window chrome.
**Critical:** Keep aria-label and title props — they pass through to the underlying button element.
**Critical:** Do NOT migrate the main nav items (lines ~244-286) — they have badge logic, active state, collapse/expand modes. Too complex for Phase 33.
