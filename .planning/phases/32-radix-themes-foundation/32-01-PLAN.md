**Key facts about @radix-ui/themes CSS:**
- Import path: `@radix-ui/themes/styles.css`
- Must load BEFORE `@tailwind base` — Tailwind's preflight and Radix Themes both set base styles; Tailwind should win for utilities but Radix needs its vars defined first
- Does NOT conflict with our --mission-control-* CSS vars (different namespace)
- Radix Themes vars use: --accent-1 through --accent-12, --gray-1 through --gray-12, --color-background, --color-surface, --color-panel-solid, --color-panel-translucent, --radius-*, --shadow-*, --font-size-*
- Our design-tokens.css uses --mission-control-* vars — zero namespace collision
</context>

```css
/* Import accessibility styles FIRST */
@import './accessibility.css';
...
```

IMPORTANT: Do NOT add it after @tailwind base — the Radix Themes vars need to be defined before Tailwind's utilities consume them, and Tailwind's base layer would reset the Radix Themes element styles.</action>
