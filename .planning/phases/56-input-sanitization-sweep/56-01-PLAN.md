<objective>
Add length limits to user-supplied string inputs that touch the DB: task title (500 chars), task description (5000 chars), agent name (100 chars), and library folder/file name (255 chars). Return 400 with descriptive error when limits are exceeded.

Purpose: Prevent oversized strings from being stored in the DB or written to disk — defends against DoS via large payloads and keeps DB fields predictable.
Output: `tasks/route.ts`, `tasks/[id]/route.ts`, `agents/hire/route.ts`, and `library/route.ts` all return 400 with a descriptive error when inputs exceed their respective limits.
</objective>

<task type="auto">
  <name>Task 1: Add length limits to task POST (create) and PATCH (update)</name>
  <files>app/api/tasks/route.ts, app/api/tasks/[id]/route.ts</files>
  <action>
In `app/api/tasks/route.ts` POST handler, after destructuring `body`, add length validation before the INSERT:

```ts
if (title && typeof title === 'string' && title.length > 500) {
  return NextResponse.json({ error: 'title must be 500 characters or fewer' }, { status: 400 });
}
if (description && typeof description === 'string' && description.length > 5000) {
  return NextResponse.json({ error: 'description must be 5000 characters or fewer' }, { status: 400 });
}
```

Also validate `title` is required (if not already validated): if `!title || typeof title !== 'string' || title.trim() === ''`, return 400 `{ error: 'title is required' }`. Place this BEFORE the length check.

In `app/api/tasks/[id]/route.ts` PATCH handler, add length validation right after parsing `body` and before building `setClauses`. Check `body.title` and `body.description` only if they are present in the body (PATCH is partial update):

```ts
if ('title' in body && (typeof body.title !== 'string' || body.title.length > 500)) {
  return NextResponse.json({ error: 'title must be 500 characters or fewer' }, { status: 400 });
}
if ('description' in body && body.description !== null && typeof body.description === 'string' && body.description.length > 5000) {
  return NextResponse.json({ error: 'description must be 5000 characters or fewer' }, { status: 400 });
}
```

Do NOT add a required check for PATCH — `title` is optional in updates.

<task type="auto">
  <name>Task 2: Add length limits to agent hire (name) and library route (folder/file name)</name>
  <files>app/api/agents/hire/route.ts, app/api/library/route.ts</files>
  <action>
In `app/api/agents/hire/route.ts` POST handler, after the `validateAgentId(id)` guard and before any filesystem work, add name length validation:

```ts
if (typeof name !== 'string' || name.trim().length === 0) {
  return NextResponse.json({ error: 'name is required' }, { status: 400 });
}
if (name.length > 100) {
  return NextResponse.json({ error: 'name must be 100 characters or fewer' }, { status: 400 });
}
```

The existing check `if (!id || !name)` already handles missing name — replace it or place before it to make the check consistent. The new check should come RIGHT AFTER the `validateAgentId` guard so the order is: (1) id/name presence check → (2) id format validate → (3) name length validate.

In `app/api/library/route.ts` POST handler, in the `folder-create` action block, add name length check before the INSERT:

```ts
if (!name || typeof name !== 'string' || name.length > 255) {
  return NextResponse.json({ success: false, error: 'folder name must be 255 characters or fewer' }, { status: 400 });
}
```

Also in the `update` action block, when `name` is in the allowed fields and present in the update body, it will pass through via the dynamic `setClauses`. Add a check before building the `updates` array:

```ts
if (typeof fields.name === 'string' && fields.name.length > 255) {
  return NextResponse.json({ success: false, error: 'name must be 255 characters or fewer' }, { status: 400 });
}
```

Place this right after `const { id, ...fields } = body;` in the `update` block.
