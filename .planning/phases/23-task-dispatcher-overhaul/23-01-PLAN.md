<objective>
Fix the task dispatcher so agents dispatched via the task board run with their full soul file context, correct per-agent model, proper MCP access via settings.json, and session continuity via --resume.

Purpose: Currently dispatched tasks spawn agents with no soul file (no identity, no GSD training, no memory protocol), hardcoded model (always sonnet regardless of agent config), and a cwd that doesn't pick up .claude/settings.json (so MCP tools are unavailable). Output: Updated `src/lib/taskDispatcher.ts` where dispatched agents have their SOUL.md as --system-prompt, correct model from the agents table, cwd set to project root for MCP access, --resume for session continuity, session_id captured and persisted, and exit code logged to task_activity.

  <files>src/lib/taskDispatcher.ts</files>
  <action>
Add session continuity so subsequent dispatches to the same agent reuse context.

1. **Session lookup**: Add `loadTaskSession(agentId: string): string | null` function:
   ```
   function loadTaskSession(agentId: string): string | null {
     try {
       const row = getDb().prepare(
         'SELECT sessionId, lastActivity FROM agent_sessions WHERE agentId = ? AND status = ?'
       ).get(agentId + ':task', 'active') as { sessionId: string; lastActivity: number } | undefined;
       if (!row?.sessionId) return null;
       // Expire sessions older than 2 hours (task sessions are shorter-lived than chat)
       if (Date.now() - row.lastActivity > 2 * 60 * 60 * 1000) return null;
       return row.sessionId;
     } catch { return null; }
   }
   ```
   Note: use `agentId + ':task'` as the key to avoid colliding with chat sessions in agent_sessions (which uses agentId as PK).

   Wait — agent_sessions has agentId as PRIMARY KEY. Using agentId + ':task' as the key means we'd have a separate row. That's fine — the table accepts any string as agentId.

2. **Session persist**: Add `persistTaskSession(agentId: string, sessionId: string, model: string)` function:
   ```
   function persistTaskSession(agentId: string, sessionId: string, model: string) {
     try {
       const now = Date.now();
       getDb().prepare(`
         INSERT OR REPLACE INTO agent_sessions (agentId, sessionId, model, createdAt, lastActivity, status)
         VALUES (?, ?, ?, COALESCE((SELECT createdAt FROM agent_sessions WHERE agentId = ?), ?), ?, 'active')
       `).run(agentId + ':task', sessionId, model, agentId + ':task', now, now);
     } catch { /* non-critical */ }
   }
   ```

3. **Use session in spawn**: In dispatchTask(), before building args:
   ```
   const existingSession = loadTaskSession(agentId);
   const args = [ '--print', '--output-format', 'stream-json', '--verbose', '--model', model, '--dangerously-skip-permissions' ];
   if (existingSession) {
     args.push('--resume', existingSession);
     // Don't add --system-prompt when resuming — session already has context
   } else {
     const systemPrompt = buildTaskSystemPrompt(agentId);
     if (systemPrompt) args.push('--system-prompt', systemPrompt);
   }
   ```

4. **Parse stdout for session_id**: After spawn, set up stdout listener:
   ```
   let outBuf = '';
   proc.stdout.on('data', (data: Buffer) => {
     outBuf += data.toString();
     const lines = outBuf.split('\n');
     outBuf = lines.pop() ?? '';
     for (const line of lines) {
       try {
         const parsed = JSON.parse(line.trim()) as { type?: string; session_id?: string };
         if (parsed.type === 'result' && parsed.session_id) {
           persistTaskSession(agentId, parsed.session_id, model);
         }
       } catch { /* not JSON, ignore */ }
     }
   });
   ```

5. Keep `proc.unref()` — the stdout listener will still fire while Next.js is running, but Node.js won't block exit waiting for the process.

6. Add imports at top: `readFileSync` from 'fs' (add to existing `existsSync` import).

Currently dispatch logs "Task dispatched to {agent}" but never logs outcome. Add exit handlers:

1. **On process close**: Listen for 'close' event and log outcome:
   ```
   proc.on('close', (code) => {
     try {
       const db = getDb();
       const exitMsg = code === 0
         ? `Agent ${agentId} completed task dispatch (exit 0)`
         : `Agent ${agentId} exited with code ${code}`;
       db.prepare(
         `INSERT INTO task_activity (taskId, agentId, action, message, timestamp) VALUES (?, ?, ?, ?, ?)`
       ).run(taskId, agentId, 'dispatch_exit', exitMsg, Date.now());

       // If non-zero exit and task is still 'todo' or 'in-progress', update to blocked
       if (code !== 0) {
         const current = db.prepare('SELECT status FROM tasks WHERE id = ?').get(taskId) as { status: string } | undefined;
         if (current && (current.status === 'todo' || current.status === 'in-progress')) {
           db.prepare(
             `UPDATE tasks SET status = 'blocked', lastAgentUpdate = ? WHERE id = ?`
           ).run(`Dispatch process exited with code ${code}. Check logs.`, taskId);
         }
       }
     } catch { /* non-critical */ }
   });
   ```

2. **On process error**: Listen for 'error' event (e.g., ENOENT if claude binary not found):
   ```
   proc.on('error', (err) => {
     console.error(`[taskDispatcher] Spawn error for task ${taskId}:`, err);
     try {
       getDb().prepare(
         `INSERT INTO task_activity (taskId, agentId, action, message, timestamp) VALUES (?, ?, ?, ?, ?)`
       ).run(taskId, agentId, 'dispatch_error', `Spawn failed: ${err.message}`, Date.now());
       getDb().prepare(
         `UPDATE tasks SET status = 'blocked', lastAgentUpdate = ? WHERE id = ?`
       ).run(`Could not start agent: ${err.message}`, taskId);
     } catch { /* non-critical */ }
   });
   ```
