<objective>
Reliable scheduling with execution history and scheduled_items processing.

Purpose: All cron executions are tracked in DB, scheduled content items fire at their time.
Output: Cron execution history in automation_runs, scheduled_items processing in cron daemon.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Add execution history logging to cron daemon</name>
  <files>tools/cron-daemon.js</files>
  <action>
  After each job execution (both runTaskJob and runMessageJob):
  1. Open SQLite DB directly using better-sqlite3 (already available as dependency)
  2. INSERT into automation_runs: id=uuid, automationId=job.id, status=success/failed, message=result summary, startedAt, completedAt
  3. On job failure, log status='failed' with error message
  4. Keep last 100 runs per job (delete older on insert)
  Also update the automations table last_run timestamp if the job corresponds to an automation.
  </action>
  <verify>node -e "require('./tools/cron-daemon.js')" does not crash on import (syntax check)</verify>
  <done>Every cron execution creates a run record in automation_runs.</done>
</task>

<task type="auto">
  <name>Task 2: Add scheduled_items processing to cron daemon</name>
  <files>tools/cron-daemon.js</files>
  <action>
  Add a new pass in the main check loop:
  1. Query scheduled_items WHERE status='pending' AND scheduledFor <= now
  2. For each pending item:
     - If type='post': create a task for the social-manager agent to post
     - If type='meeting': post a reminder to mission-control chat
     - If type='event': log to calendar activity
  3. Update status to 'completed' after processing
  Run this check every minute alongside the existing schedule.json check.
  </action>
  <verify>node -e "require('./tools/cron-daemon.js')" does not crash on import</verify>
  <done>Pending scheduled_items are processed at their scheduled time.</done>
</task>

</tasks>
