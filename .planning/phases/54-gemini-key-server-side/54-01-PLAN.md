<objective>
Stop the Gemini API key from being written to localStorage or bundled into the client via NEXT_PUBLIC_ env var. Fix the two root causes: (1) the settings API route uses NEXT_PUBLIC_GEMINI_API_KEY which bakes the key into the client bundle, and (2) SettingsPanel writes geminiApiKey into the mission-control-settings localStorage blob.

Purpose: The key is already fetched server-side via settingsApi.get('gemini_api_key') in all components — the remaining exposure is from localStorage persistence (key survives across browser sessions in plaintext) and the NEXT_PUBLIC_ env var (key baked into the JS bundle). Removing both closes these two exposure vectors.
Output: settings/[key]/route.ts uses GEMINI_API_KEY (server-only). SettingsPanel excludes geminiApiKey from localStorage, saves it directly to DB via settingsApi, loads it from DB on mount.
</objective>

Three changes needed in SettingsPanel.tsx:

**1. Initial state load — don't load geminiApiKey from localStorage:**
The `useState` initializer reads from localStorage. Change it so geminiApiKey is always initialized empty (will be loaded from DB separately):

```ts
// Find the useState(() => { ... }) initializer:
const [settings, setSettings] = useState<Settings>(() => {
  const saved = localStorage.getItem('mission-control-settings');
  return saved ? { ...defaultSettings, ...JSON.parse(saved) } : defaultSettings;
});
```
Change to:
```ts
const [settings, setSettings] = useState<Settings>(() => {
  const saved = localStorage.getItem('mission-control-settings');
  const parsed = saved ? JSON.parse(saved) : {};
  // Exclude geminiApiKey from localStorage — it lives in DB only
  const { geminiApiKey: _ignored, ...rest } = parsed;
  return { ...defaultSettings, ...rest };
});
```

**2. Load geminiApiKey from DB on mount — add a useEffect after the existing theme effects:**
```ts
useEffect(() => {
  settingsApi.get('gemini_api_key').then(result => {

      setSettings(s => ({ ...s, geminiApiKey: result.value }));
    }
  }).catch(() => { /* non-critical */ });
}, []);
```
