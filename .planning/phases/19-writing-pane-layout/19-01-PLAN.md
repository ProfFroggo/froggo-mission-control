**Why these specific values:**
- `minSize="180px"` — string with px unit tells the library to use pixels. 180px fits ~12 chars of chapter titles.
- `maxSize="30%"` — string with % tells the library to use percentage. Prevents chapters from consuming too much space.
- `minSize="280px"` — chat needs ~280px minimum to render the input bar and messages legibly.
- `maxSize="50%"` — chat can grow to half the window.
- `minSize="300px"` — editor needs at least 300px for comfortable writing.
- `w-2` (8px) — wider hit target than `w-1` (4px), making the handle grabbable.
- Double `w-px h-5` divs — visual double-line grip indicator, standard resize handle pattern.
- `data-[separator=active]:` variant — provides drag feedback throughout the entire drag lifecycle. The library sets `data-separator="active"` on the Separator element during drag. **Do NOT use `active:` CSS pseudo-class** — it drops when the cursor moves off the element mid-drag. Old v3 attribute `data-[resize-handle-active]` does not exist in v4.6.2.
- `group` + `group-hover`/`group-data-[separator=active]` — child grip lines respond to parent hover/active-drag state via Tailwind's arbitrary group-data variant.

With:
