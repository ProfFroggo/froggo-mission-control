**Origins used by the app:**
- REST: `https://generativelanguage.googleapis.com` (Gemini API — MeetingsPanel, MeetingScribe, VoiceChatPanel, QuickActions)
- WebSocket: `wss://generativelanguage.googleapis.com` (Gemini Live — geminiLiveService.ts, MeetingsPanel)
- `blob:` required for audio recording and video streams (MediaRecorder, WebAudio)
- `data:` for base64 audio/images
- Next.js App Router requires `'unsafe-inline'` for style-src (Tailwind inline styles) and `'unsafe-eval'` for script-src (dev HMR; acceptable for local-only app)

**Constraining decisions:**
- App runs locally (localhost:3000) — pragmatic CSP with 'unsafe-inline'/'unsafe-eval' is appropriate; goal is to block external framing/injection, not restrict own trusted scripts
- No external CDN fonts or scripts — all assets are self-hosted

<task type="auto">
  <name>Task 1: Add security headers to next.config.js via headers()</name>
  <files>next.config.js</files>
  <action>
Replace the simple `module.exports = nextConfig` with a version that adds a `headers()` async function returning an array with one rule (source: '/(.*)', all routes):

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  serverExternalPackages: ['better-sqlite3'],

  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          { key: 'X-Frame-Options',           value: 'DENY' },
          { key: 'X-Content-Type-Options',     value: 'nosniff' },
          { key: 'Referrer-Policy',            value: 'strict-origin-when-cross-origin' },
          { key: 'X-XSS-Protection',           value: '1; mode=block' },
          {
            key: 'Content-Security-Policy',
            value: [
              "default-src 'self'",
              "script-src 'self' 'unsafe-eval' 'unsafe-inline'",
              "style-src 'self' 'unsafe-inline'",
              "connect-src 'self' https://generativelanguage.googleapis.com wss://generativelanguage.googleapis.com ws://127.0.0.1:*",
              "img-src 'self' data: blob:",
              "media-src 'self' blob:",
              "worker-src 'self' blob:",
              "font-src 'self'",
              "frame-ancestors 'none'",
            ].join('; '),
          },
        ],
      },
    ];
  },
};

module.exports = nextConfig;
```

`ws://127.0.0.1:*` in connect-src covers the local voice bridge WebSocket (port 8765) and gateway (port 18789). `blob:` in media-src and worker-src is required by the MediaRecorder API used in meeting/voice components. `frame-ancestors 'none'` in CSP is redundant with X-Frame-Options but provides defense-in-depth for browsers that prefer CSP.

Do NOT add `upgrade-insecure-requests` — app runs on HTTP locally.
Do NOT set `Strict-Transport-Security` — app is HTTP localhost, HSTS on localhost causes browser issues.

const c = require('./next.config.js');

  const headers = h[0].headers;
  const keys = headers.map(hh => hh.key);
  const required = ['X-Frame-Options','X-Content-Type-Options','Referrer-Policy','X-XSS-Protection','Content-Security-Policy'];
  const missing = required.filter(r => !keys.includes(r));
  if (missing.length) { console.error('MISSING:', missing); process.exit(1); }
  const csp = headers.find(hh => hh.key === 'Content-Security-Policy').value;
  const reqOrigins = ['generativelanguage.googleapis.com', 'wss://generativelanguage.googleapis.com', \"frame-ancestors 'none'\"];

The static verification is sufficient since Next.js header config is declarative — if headers() returns correct data, Next.js will serve them.
