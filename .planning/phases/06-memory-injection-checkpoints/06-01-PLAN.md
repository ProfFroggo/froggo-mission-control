<objective>
Agents save learnings after task completion and old memories are summarized to stay under size limits.

Purpose: Memory accumulates over time — agents get smarter with each task.
Output: Auto-checkpoint on task completion, memory size management.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Session checkpoint on task completion</name>
  <files>src/lib/claraReviewCron.ts</files>
  <action>
  In the post-review handler, when Clara approves a task (status=done, reviewStatus=approved):
  1. Read the task's activity log (last 10 entries), subtasks, attachments
  2. Create a checkpoint file at ~/mission-control/agents/{assignedTo}/memory/{date}-{task-title-slug}.md
  3. Contents: YAML frontmatter (date, task, agent, tags), then: what was done, key decisions, files created, lessons learned (extracted from activity log)
  4. Cap at 500 chars to keep memory lean

  This happens AFTER Clara approves, so the checkpoint captures the full task lifecycle.
  </action>
  <verify>npx tsc --noEmit passes</verify>
  <done>Completed tasks auto-generate checkpoint files in agent memory dir.</done>
</task>

<task type="auto">
  <name>Task 2: Memory size management</name>
  <files>src/lib/claraReviewCron.ts</files>
  <action>
  Add a memory housekeeping pass to runReviewCycle() (runs every 3 min):
  1. For each agent dir with >30 memory files, archive oldest files (move to memory/archive/)
  2. Keep the 20 most recent files active
  3. Only run this check once per hour (use timestamp sentinel)
  </action>
  <verify>npx tsc --noEmit passes</verify>
  <done>Agent memory dirs stay under 30 files with oldest archived.</done>
</task>
