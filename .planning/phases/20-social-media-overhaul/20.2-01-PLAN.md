# Plan: Pipeline Multi-View + Calendar/List

## Objective

Add view mode toggle to Pipeline tab: Board (existing kanban) | Calendar | List. Board is default. Calendar absorbs XCalendarView logic. List absorbs XDraftListView filtering. Plan items render as kanban cards (no separate tab).

## Execution Context

@src/components/XPipelineView.tsx (527 lines — the kanban board)
@src/components/XCalendarView.tsx (181 lines — calendar event mapping)
@src/components/XDraftListView.tsx (360 lines — filtered list with approve/reject)
@src/components/XPlanListView.tsx (137 lines — read-only plan list)

## Tasks

### Task 1: Add view mode toggle to XPipelineView

**Requirements:**
- Add state: `viewMode: 'board' | 'calendar' | 'list'` (default: 'board')
- Toggle bar below header: 3 buttons with icons (Columns, Calendar, List)
- Active mode highlighted with `bg-info-subtle text-info`
- Board view: existing kanban (unchanged)
- Ensure `type='plan'` items appear in the kanban alongside drafts/ideas
- Remove plans from being a separate concern — they're just another pipeline item type with a 'plan' badge

### Task 2: Build Calendar view mode

Inline the calendar view within XPipelineView.

**Requirements:**
- Reuse XCalendarView's `mapResearchToEvents` + `mapScheduledToEvents` logic
- Render EpicCalendar within the Pipeline component when viewMode='calendar'
- Same color coding: purple=research, blue=plan, amber=draft, emerald=scheduled
- Add click-on-event to open a detail popover (show full content, status, actions)
- "Create Tweet" button still available (dispatches x-tab-change or opens compose modal)

### Task 3: Build List view mode

Merge XDraftListView's functionality as the list view.

**Requirements:**
- Reuse filter pills: All | Ideas | Drafts | Approved | Rejected | Scheduled | Published
- Table/list layout: each row shows content preview, type badge, status badge, agent, date, actions
- Actions per row: same as kanban card actions (Draft, Submit, Approve, Reject, Schedule, Publish)
- "Approve all" bulk action (from XDraftListView)
- Sortable by date (newest first by default)
- Search/filter input for content text
- Design: clean table with `divide-y divide-mission-control-border`, hover highlight
