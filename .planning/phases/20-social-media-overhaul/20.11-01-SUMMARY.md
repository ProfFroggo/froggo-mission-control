# Summary: Automation Builder Upgrade

## Objective
Upgrade the automation builder with AI engine selector, new action types, execution history UI, and conversational creation via agent chat.

## What Was Done

### Task 1: AI Engine Selector + New Action Types
**Files changed**: `src/lib/database.ts`, `app/api/x/automations/route.ts`, `app/api/x/automations/execute/route.ts`, `src/components/XAutomationsTab.tsx`

- Added `ai_engine` column to `x_automations` table via ALTER TABLE migration (safe to run on every startup, silently ignored if column exists)
- Added AI engine selector UI in the builder between trigger and actions — two large toggle cards: "Gemini Flash Lite (fast, cheap)" and "Claude Haiku (interactive, nuanced)"
- Added engagement trigger config UI: min_likes and min_retweets number inputs (shown only when engagement trigger selected)
- Added action config UI for `report` (report_type select: competitor-analysis | weekly-summary), `post_content` (content template textarea), and `custom_prompt` (prompt textarea)
- Updated `getDefaultActionConfig` with sensible defaults for report, post_content, custom_prompt
- Persisted `ai_engine` in POST (INSERT) and PATCH (UPDATE) automations API handlers
- Updated execute/route.ts: `aiEngine` passed into `executeAction`; for `reply` actions with `ai_engine='claude'`, calls `/api/chat/generate-reply` to craft a contextual reply from the mention text instead of using a static template
- Builder correctly restores `ai_engine` when editing existing automations

### Task 2: Execution History UI + Conversational Creation
**Files changed**: `src/components/XAutomationsTab.tsx`, `src/components/XAgentChatPane.tsx`

- Replaced flat execution log list with enhanced surface card:
  - Collapsible header with total count
  - Filter input: type to filter entries by automation name in real-time
  - Each entry shows: timestamp, automation name (resolved from automations list), trigger type, actions taken, status badge
  - Collapsible detail row per entry: automation ID, each action's type/status/approval_id, error text if present
  - "Load more" button when more than 20 entries exist (shows 20 at a time)
  - Resets display limit when filter changes
- Added state: `logFilter`, `logDisplayLimit`, `expandedLogEntry`
- "Create with AI" button (already existed) dispatches `x-agent-chat-inject` event with pre-filled prompt
- `x-automation-created` listener (already existed) reloads automations list and shows toast
- Updated `XAgentChatPane` configure tab system context with full list of new trigger/action types and `ai_engine` field — agent now produces valid automation JSON blocks for all new types
