# Plan: Adaptive Layout + Floating Compose

## Objective

Replace the rigid XThreePaneLayout (always-on 3 columns: chat 30% | content 40% | approvals 30%) with an adaptive layout where content gets full width by default and agent chat is a slide-in toggle panel. Move XPublishComposer from a tab into a floating modal/drawer accessible from any tab.

## Context

- XThreePaneLayout.tsx (157 lines): Drag-resizable 3 columns with collapse toggles
- XTwitterPage.tsx (166 lines): Routes tabs, controls which tabs show approval pane

- XAgentChatPane.tsx (610 lines): Per-tab agent chat with quick prompts, message history

Current tab type: `'pipeline' | 'publish' | 'research' | 'plan' | 'drafts' | 'calendar' | 'mentions' | 'reply-guy' | 'content-mix' | 'automations' | 'analytics' | 'campaigns' | 'agent-mode' | 'competitors' | 'hashtags'`

New tab type: `'pipeline' | 'engage' | 'intelligence' | 'measure' | 'configure'`

## Tasks

### Task 1: Build XSocialLayout

Create `src/components/XSocialLayout.tsx` — the new adaptive layout.

**Structure:**
```
+------------------------------------------------------------------+
| Header: [tabs] ..................... [approval badge] [chat] [+]  |
+------------------------------------------------------------------+
|                                                    |              |
|  MAIN CONTENT (100% by default)                    | Agent Chat   |
|  {children}                                        | 380px fixed  |
|                                                    | slide-in     |
+------------------------------------------------------------------+
```

**Requirements:**
- Main content area: `flex-1` fills remaining width
- Agent chat panel: 380px fixed width, slides in from right with transition
- Toggle button in header (message icon + "Agent" label)
- Chat state remembered per tab (open/closed stored in local state)
- No drag-resize — fixed width when open, gone when closed
- Smooth CSS transition on open/close (transform or width animation, ~200ms)
- Header contains: tab bar, approval badge area, chat toggle, compose button
- Remove all XThreePaneLayout references from XTwitterPage
- Design tokens: all borders use `border-mission-control-border`, all backgrounds use `bg-mission-control-surface` or `bg-mission-control-bg`

**Agent chat panel styling:**
- Right border: `border-l border-mission-control-border`
- Background: `bg-mission-control-surface`
- Header with agent name + close button
- Full height minus header

### Task 2: Build XComposeModal

Create floating compose as a modal/drawer that renders XPublishComposer.

**Requirements:**
- `[+ Compose]` button in XSocialLayout header (always visible, any tab)
- Opens a right-side drawer or centered modal (drawer preferred — less jarring)
- Renders existing XPublishComposer inside (no rewrite needed)
- Backdrop overlay with click-to-close
- Escape key closes
- Keyboard shortcut: `Cmd+N` or `Cmd+Shift+C` opens compose
- Remove 'publish' from tab bar (it's now the floating compose)
- z-index above chat panel
- Width: `max-w-2xl` (672px) for comfortable compose experience

### Task 3: Rewire XTwitterPage

Update the main page to use new layout + new tab type.

**Requirements:**
- Replace XThreePaneLayout with XSocialLayout
- Update XTab type to: `'pipeline' | 'engage' | 'intelligence' | 'measure' | 'configure'`
- Remove TABS_WITH_APPROVAL (no more conditional approval pane)
- Update XTabBar to 5 tabs with new icons
- Default tab: 'pipeline'
- Remove XApprovalQueuePane import (will be replaced by badge in next plan)
- Remove XContentEditorPane — route directly in layout based on activeTab
- Keep setup wizard check logic intact
- Keep scheduled/pending count loading for the approval badge
