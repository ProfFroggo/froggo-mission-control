**Fix: Competitor cards stats loading issue**

File: `src/components/XCompetitorTracker.tsx`

Root cause: On mount, `loading` initializes to `false` even when handles exist, so before `fetchCompetitors()` fires `setLoading(true)`, the UI briefly shows the "Tracked accounts" chips section (because `handles.length > 0 && competitors.length === 0 && !loading` is true). More importantly, the user profile data (`data.user?.public_metrics`) is frequently `null` because the X search API's `includes.users` expansion only returns users who authored tweets in the result set — if the competitor's username is not in those results, the user object won't be populated, leaving the followers count missing.

Fixes to apply:
1. Initialize `loading` state to `true` if handles exist on mount (prevents flicker): `useState(() => loadHandles().length > 0)`
2. Add `isPartialData` visual hint to the competitor card when `data.user === null` — a small badge showing "Partial data – follower stats unavailable" in the card header subtitle area, styled with `text-mission-control-text-dim` and a `HelpCircle` icon from lucide-react
3. Ensure the loading state is properly reset if handles is empty on mount
</task>
