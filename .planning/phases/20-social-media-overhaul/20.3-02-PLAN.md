# Plan: Engage View — Reply Composer + Approval Gate

## Objective

Add inline reply composer to XEngageView with template support, fast-track approval toggle, and approval gate. All replies go through approvalApi — no direct posting bypass.

## Execution Context

@src/components/XEngageView.tsx (built in 20.3-01)
@src/components/XReplyGuyView.tsx (reply templates, fast-track, approval)
@src/components/XMentionsView.tsx (reply composer, approval)
@src/lib/api.ts (approvalApi)

## Tasks

### Task 1: Inline reply composer

**Requirements:**
- Click "Reply" on a mention card → expands inline reply section below the card
- Textarea with placeholder "Write your reply..."
- Character counter: `{length}/280`
- Auto-resize textarea (2-4 rows based on content)
- Cancel button collapses reply section
- Only one reply open at a time (clicking Reply on another card closes the previous)

### Task 2: Reply templates + fast-track

**From XReplyGuyView:**
- Template grid: clickable template cards that populate reply textarea
- Default templates: Thank you, Acknowledgment, Question, Support
- Custom templates: stored in localStorage `x-reply-templates`
- Template variable substitution: `{{username}}`, `{{topic}}`
- Fast-track toggle: checkbox "Fast-track (tier 1 review)"
- When fast-track is on, approval tier = 1 (quicker review)
- When off, tier = 3 (standard review)
- "Suggest Reply" button → dispatches to agent chat: `x-agent-chat-inject` with mention context

### Task 3: Approval gate for all replies

**Requirements:**
- Send button calls `approvalApi.create({ type: 'x-reply', tier: fastTrack ? 1 : 3, payload: { mentionId, tweetId, replyText }, requestedBy: 'user' })`
- Success: show brief "Sent for approval" toast, collapse reply, clear text
- Error: show error inline below textarea
- NO direct posting to `/api/x/tweet` — remove the bypass that existed in XReplyGuyView
- After approval creation, reload mentions to reflect updated state
