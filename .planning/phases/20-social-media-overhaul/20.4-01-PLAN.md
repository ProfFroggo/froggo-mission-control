# Plan: Intelligence View — Sub-Tab Wrapper

## Objective

Create XIntelligenceView that wraps Research, Competitors, and Hashtags as sub-tabs. The existing components stay largely intact — this is a container with sub-tab routing.

## Execution Context

@src/components/XResearchView.tsx (469 lines — search + library)
@src/components/XCompetitorTracker.tsx (483 lines — handle tracking + tweets)
@src/components/XHashtagIntelligence.tsx (631 lines — hashtag search + sets)

## Tasks

### Task 1: Build XIntelligenceView

Create `src/components/XIntelligenceView.tsx`.

**Requirements:**
- Sub-tab bar: `[Search] [Competitors] [Hashtags]`
- Default sub-tab: 'search'
- Sub-tab styling: smaller than main tabs, `text-sm`, underline active indicator
- Renders the existing component based on active sub-tab:
  - 'search' → XResearchView
  - 'competitors' → XCompetitorTracker
  - 'hashtags' → XHashtagIntelligence
- Sub-tab state persisted in component (not URL, not localStorage)
- Clean transition between sub-tabs (no flash/flicker)
