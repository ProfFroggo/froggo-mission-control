# Plan: Engage View — Unified Mention Inbox + Smart Filters

## Objective

Build XEngageView merging XMentionsView (402 lines) and XReplyGuyView (659 lines) into a single engagement inbox. All mentions in one stream with smart filters, priority accounts, and sentiment detection.

## Execution Context

@src/components/XMentionsView.tsx (402 lines)
@src/components/XReplyGuyView.tsx (659 lines)
@src/lib/api.ts (inboxApi, approvalApi)

## Tasks

### Task 1: Build XEngageView with unified data loading

Create `src/components/XEngageView.tsx`.

**Data loading:**
- Single data source: `inboxApi.getAll()` filtered for `type='x-mention'` (same as both former components)
- Fallback: `/api/x/mentions` if inbox empty (from XMentionsView)
- "Fetch New" button syncs X API mentions to inbox (from XMentionsView.fetchNewMentions)

**Smart filters (from both components combined):**
- All — every mention
- Hot — engagement threshold (from Reply Guy): likes >= minLikes OR retweets >= minRetweets
- Pending — reply_status='pending'
- Replied — reply_status='replied'
- Ignored — reply_status='ignored'

**Filter bar:**
- Filter pills: `[All (42)] [Hot (5)] [Pending (28)] [Replied (8)] [Ignored (6)]`
- Active filter: `bg-info-subtle text-info font-medium`
- Engagement threshold controls (from Reply Guy): collapsible row with "Min likes" and "Min retweets" sliders/inputs, only visible when "Hot" filter active

**Priority accounts (from Reply Guy):**
- Star icon on mention cards for priority accounts
- Priority mentions sorted to top
- "Priority Accounts" settings panel (collapsible in header area)
- Stored in localStorage `x-reply-priority-accounts`

**Sentiment (from Reply Guy):**
- Inline sentiment badge on each mention card (positive/neutral/negative)
- Sentiment filter option (optional, can be secondary filter)
- Uses existing naive keyword inference (can upgrade later)

### Task 2: Build mention cards with engagement metrics

**Mention card anatomy:**
```
+-----------------------------------------------------------+
| @username · Display Name     [*priority] sentiment  date  |
|                                                           |
| Tweet text here showing the full mention content...       |
|                                                           |
| [heart] 42  [retweet] 5  [reply] 3  [View on X ->]      |
|                                                           |
| Status: [Pending] [Considering] [Ignored]     [Reply ->]  |
+-----------------------------------------------------------+
```

**Requirements:**
- Author info: @username, display name, avatar if available
- Full tweet text with `whitespace-pre-wrap`
- Engagement metrics row: likes, retweets, replies + "View on X" link
- Status management: buttons to set pending/considering/ignored (from XMentionsView)
- Notes: inline input with save (from XMentionsView)
- Card styling: `rounded-xl border border-mission-control-border bg-mission-control-surface p-4 mb-3`
- Hover: `hover:bg-mission-control-bg-alt transition-colors`
- Priority indicator: gold star icon top-right
- Sentiment badge: colored pill (green/gray/red) next to date

### Task 3: Empty states and loading

**Requirements:**
- Loading state: skeleton cards or centered spinner
- Empty (no mentions): centered icon (Inbox) + "No mentions yet" + "Click Fetch New to check"
- Empty (filter active): "No [filter] mentions" + "Try a different filter"
- Fetching state: button shows spinner + "Fetching..." (from XMentionsView)
