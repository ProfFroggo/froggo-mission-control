# Plan: Unified Approval Badge

## Objective

Replace the dedicated approval queue pane with a lightweight approval badge in the header. Clicking shows a dropdown/popover listing ALL pending approvals across all types (tweets, replies, research, plans, drafts, campaigns) with quick-action buttons.

## Execution Context

@src/components/XApprovalQueuePane.tsx
@src/components/XTwitterPage.tsx
@src/lib/api.ts (approvalApi)

## Tasks

### Task 1: Build XApprovalBadge

Create `src/components/XApprovalBadge.tsx`.

**Requirements:**
- Fetches `approvalApi.getAll('pending')` on mount + polls every 30s
- Shows count badge (red dot with number) in header
- Click opens a dropdown/popover (positioned below badge, right-aligned)
- Dropdown lists all pending approvals grouped by type:
  - Tweets (type='tweet')
  - Replies (type='x-reply')
  - Research (type='research')
  - Plans (type='plan')
  - Drafts (type='draft')
- Each item shows: type icon, content preview (truncated), age, [Approve] [Reject] buttons
- Approve/reject calls `approvalApi.respond()` and removes item from list
- Empty state: "No pending approvals" with checkmark icon
- Close on click outside, Escape key
- Design: `bg-mission-control-surface border border-mission-control-border rounded-xl shadow-lg`
- Max height: `max-h-[480px] overflow-y-auto`
- Width: `w-96` (384px)
