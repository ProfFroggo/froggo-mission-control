Create three new files:

**src/lib/wizardSchema.ts:**
- Import `z` from `zod`
- Define and export `wizardChapterSchema` = z.object({ title: z.string().min(1), synopsis: z.string() })
- Define and export `wizardCharacterSchema` = z.object({ name: z.string().min(1), role: z.string(), description: z.string(), traits: z.array(z.string()) })
- Define and export `wizardTimelineSchema` = z.object({ date: z.string(), description: z.string() })
- Define and export `wizardPlanSchema` = z.object({ title: z.string().min(1), type: z.string().min(1), genre: z.string(), premise: z.string(), themes: z.array(z.string()), storyArc: z.string(), chapters: z.array(wizardChapterSchema).min(1), characters: z.array(wizardCharacterSchema), timeline: z.array(wizardTimelineSchema) })
- Export type `WizardPlan = z.infer<typeof wizardPlanSchema>`

**src/lib/wizardPrompts.ts:**
- Define and export `WIZARD_AGENTS` record with `memoir` and `novel` keys. Each has `id` (jess/writer), `name`, and `preamble` string. Use the preambles from the research (Pattern 5).
- Export `getWizardAgent(bookType: string)` — returns `WIZARD_AGENTS.memoir` if bookType === 'memoir', else `WIZARD_AGENTS.novel`
- Export `buildConversationPrompt(agent: string, brainDump: string, conversationSummary?: string): string` — builds the system prompt for conversation phase. Include the agent preamble, role instructions (ask clarifying questions, propose structure, be collaborative not prescriptive), and the user's brain dump. If conversationSummary provided, include "Decisions So Far" section.
- Export `buildExtractionPrompt(conversationSummary: string): string` — builds the extraction prompt that asks for ONLY a JSON code block matching the WizardPlan schema. Include the exact JSON structure template. Append conversation summary.
- Export `parseWizardPlan(response: string): WizardPlan | null` — regex match `\`\`\`json\s*([\s\S]*?)\`\`\`` from response, JSON.parse, then validate with `wizardPlanSchema.parse()`. Return null on any failure. Import wizardPlanSchema from wizardSchema.ts.

**src/store/wizardStore.ts:**
- Import `create` from `zustand`
- Import `WizardPlan` from `../lib/wizardSchema`
- Re-export `ChatMessage` type from `./chatPaneStore` (same type used for wizard messages)
- Define `WizardStep = 'idle' | 'braindump' | 'conversation' | 'extracting' | 'review' | 'creating' | 'complete'`
- Define WizardState interface with: step, sessionId (string|null), messages (ChatMessage[]), streaming (boolean), streamContent (string), selectedAgent (string, default 'writer'), brainDump (string), plan (WizardPlan|null), extractionError (string|null), error (string|null)
- Actions: startWizard (generates sessionId `wiz-{Date.now()}-{random}`, sets step to 'braindump'), cancelWizard (calls bridge()?.delete(sessionId), resets all state), setStep, setBrainDump, setSelectedAgent, setStreaming, setStreamContent, addMessage, setPlan, updatePlan (partial merge into existing plan), setExtractionError, setError, clearMessages, loadState (partial state merge for resume), reset
- Use `bridge = () => (window as any).clawdbot?.writing?.wizard` accessor pattern (same as writingStore)
- Export `useWizardStore`
