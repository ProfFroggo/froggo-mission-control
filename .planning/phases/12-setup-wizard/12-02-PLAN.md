**src/components/writing/WizardChat.tsx:** (NEW)
A chat interface for the wizard conversation, following the ChatPane.tsx streaming pattern.

- Import useWizardStore, gateway from gateway.ts, buildConversationPrompt/getWizardAgent/buildExtractionPrompt/parseWizardPlan from wizardPrompts
- Import ChatMessage component from ./ChatMessage and ChatInput from ./ChatInput
- Import useRef, useEffect, useCallback from react
- Use `accumulatedRef = useRef('')` for streaming accumulation (same as ChatPane)
- Use `scrollRef` for auto-scroll to bottom

- `handleSend(text: string)` function:
  1. Add user message to store via addMessage
  2. Set streaming true, streamContent to ''
  3. Build sessionKey: `agent:${getWizardAgent(selectedAgent).id}:writing:wizard:${sessionId}`
  4. Build the full message. For the FIRST message (messages.length === 0 before adding), prepend the conversation system prompt via buildConversationPrompt with the brainDump. For subsequent messages, just send the user's text.
  5. Call `gateway.sendChatWithCallbacks(message, sessionKey, { onToken, onComplete, onError })`
  6. `onToken`: append to accumulatedRef.current, call setStreamContent
  7. `onComplete`: add assistant message with full content, set streaming false, clear streamContent, reset accumulatedRef. Then persist wizard state by calling `bridge()?.save(sessionId, { step, sessionId, messages: [...messages, userMsg, assistantMsg], selectedAgent, brainDump, plan })`
  8. `onError`: set error, set streaming false

- `handleExtract()` function (called when step is 'extracting'):
  1. Build a conversation summary from messages (concatenate user + assistant content, limit to ~4000 chars focusing on decisions)
  2. Use a SEPARATE session key for extraction: `agent:${agentId}:writing:wizard-extract:${sessionId}`
  3. Send buildExtractionPrompt(summary) via sendChatWithCallbacks
  4. On complete: call parseWizardPlan(response). If valid, setPlan(result), setStep('review'). If null, setExtractionError('Failed to parse plan'), setStep('conversation') so user can retry.
  5. On error: setExtractionError(message), setStep('conversation')

- useEffect: if step === 'extracting', call handleExtract()

- Auto-send first message: useEffect that fires when messages are empty and step is 'conversation'. Automatically send the brain dump as the first user message (triggers AI to respond to the idea).

- Render: scrollable message list using ChatMessage component, streaming indicator, ChatInput at bottom. Show extraction error banner if extractionError is set. During 'extracting' step, show a loading overlay "Generating your book plan..."

- The "Generate Plan" button in SetupWizard header triggers setStep('extracting') which this component detects and runs extraction.
