**FUNC-10: Share a single debounced task refresh**

In `store.ts`, just above the gateway.on listeners (around line 1265), add a shared debounced refresh function:

```typescript
// Shared debounced task refresh â€” used by both gateway.on and IPC broadcast
const TASK_REFRESH_DEBOUNCE = 400;
function debouncedTaskRefresh() {
  clearTimeout((window as any).__taskRefreshTimer);
  (window as any).__taskRefreshTimer = setTimeout(() => {
    useStore.getState().loadTasksFromDB();
  }, TASK_REFRESH_DEBOUNCE);
}
```

Then simplify ALL task event listeners to use it:

```typescript
gateway.on('task.created', (payload: any) => {
  console.log('[Store] Task created event received:', payload);
  debouncedTaskRefresh();
  useStore.getState().addActivity({
    type: 'task',
    message: `New task: ${payload?.title || 'Task created'}`,
    timestamp: Date.now(),
  });
});

gateway.on('task.updated', (payload: any) => {
  console.log('[Store] Task updated event received:', payload);
  debouncedTaskRefresh();
});

gateway.on('tasks.refresh', () => {
  console.log('[Store] Tasks refresh event received');
  debouncedTaskRefresh();
});
```

And the broadcast listener:

```typescript
if (typeof window !== 'undefined' && (window as any).clawdbot?.gateway?.onBroadcast) {
  (window as any).clawdbot.gateway.onBroadcast((data: { type: string; event: string; payload: any }) => {
    console.log('[Store] Gateway broadcast received:', data.event, data.payload?.id);
    if (data.event === 'task.created' || data.event === 'task.updated') {
      debouncedTaskRefresh();
    }
  });
}
```

Both listener paths now call the SAME `debouncedTaskRefresh()` with the SAME timer key (`__taskRefreshTimer`), so rapid events from either source collapse into a single 400ms-debounced `loadTasksFromDB()` call.

**FUNC-08: Add custom memo comparator to TaskCard (Kanban.tsx line 984)**

The current `memo(function TaskCard(...))` has no custom comparator. Add one that checks the fields that affect rendering, intentionally skipping callback props (which are new references each render but don't affect visual output):

Find the closing of the TaskCard component function body (the final `}` before the `)` that closes `memo()`). Add a comparator as second argument to `memo()`:

```typescript
const TaskCard = memo(function TaskCard({ task, agents, activeSessions, onDragStart, onDragEnd, onDelete, onAssign, onStartAgent, onClick, onSetPriority, onPoke, isDragging, isDeleting, isSpawning, isMoving }: TaskCardProps) {
  // ... entire component body unchanged ...
}, (prev, next) => {
  // Return true if props are equal (skip re-render)
  // Intentionally skip callback comparison â€” they're recreated each render but don't affect visual output
  return (
    prev.task === next.task &&
    prev.isDragging === next.isDragging &&
    prev.isDeleting === next.isDeleting &&
    prev.isSpawning === next.isSpawning &&
    prev.isMoving === next.isMoving &&
    prev.agents === next.agents &&
    // activeSessions is a new object each render â€” deep compare
    JSON.stringify(prev.activeSessions) === JSON.stringify(next.activeSessions)
  );
});
```

The key insight: `activeSessions` is `Record<string, boolean>` and gets a new reference every time `setActiveSessions` runs (even with identical data), so shallow `===` always fails. `JSON.stringify` comparison handles this cheaply for small objects.
