## Milestone Summary

**Key Decisions:**

- CSS token pattern: define in `src/index.css` (`:root` + `:root.light`), expose in `tailwind.config.js` with `var()` + hex fallback
- User chat bubbles: `bg-clawd-accent/50 text-white` across ALL chat components
- `TABS_WITH_APPROVAL` allowlist as source of truth for approval panel visibility
- `eventColorResolver` callback pattern for custom calendar event colors
- `externalEvents` prop pattern for external calendar data (bypasses DB fetch)
- `autoSend` + `CustomEvent` pattern for cross-component chat injection
- Idempotent `ALTER TABLE` per-column with try/catch for schema migrations
- `data-[separator=active]` CSS selector for react-resizable-panels v4 drag state
- `agent_skills` table (not `skill_evolution`) as skills source of truth
- `finance_alerts` uses `acknowledged` column (0/1), NOT `status` column
- PDF upload delegates to finance-manager agent (fire-and-forget, no await on response)

**Issues Resolved:**

- All finance CLI commands lacked `--format json` — silent JSON.parse failures in every finance IPC handler
- `finance-alerts` crashed on non-existent `status` column (uses `acknowledged` 0/1)
- Finance upload IPC omitted required `--account` and `--budget-type` CLI params
- Phase 20 PR merge conflict on `.planning/` files resolved with `git checkout --theirs`
- LibraryFile type collision between global `global.d.ts` and local component interface

**Issues Deferred:**

- `gateway.ts sendChat()`: competing content accumulation (line ~483 replaces, line ~495 appends) — known race
- `electron/main.ts`: 7K+ line monolith with 60+ IPC handlers
- Pre-existing TypeScript type mismatches in finance files (history/messages, response/message field names)

**Technical Debt Incurred:**

- finance IPC handlers still use wrong field names for `history`/`response` — runtime works but TS types diverge
- `finance:uploadCSV` hardcodes `acc-default`/`family` — no per-upload account selection
