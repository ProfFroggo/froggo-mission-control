# Milestone v2.1: Writing UX Redesign

**Status:** SHIPPED 2026-02-13
**Phases:** 11-12
**Total Plans:** 8

## Overview

Transform the writing module into an AI-powered book creation system with conversational AI chat driving content into the editor, and a setup wizard that plans the book before writing begins.

## Phases

### Phase 11: Chat Pane + 3-Pane Layout

**Goal**: User can chat with AI agents in a persistent pane alongside the editor, within a resizable 3-pane layout, and insert AI-generated content into the editor
**Depends on**: Phase 10 (uses v2.0 writing infrastructure: TipTap editor, gateway streaming, agent sessions, memory store)
**Requirements**: CHAT-01 through CHAT-10, LAYOUT-01 through LAYOUT-05
**Plans**: 4 plans

Plans:
- [x] 11-01: Chat stores, pendingInsert mechanism, shared context utility, chat history IPC
- [x] 11-02: Install npm packages, 3-pane resizable layout, CSS
- [x] 11-03: ChatPane/ChatMessage/ChatInput components, gateway streaming, editor insertion
- [x] 11-04: Session key fix, copy/retry actions, persistence verification

**Success Criteria:**
1. User can open the writing workspace and see a 3-pane layout with drag-resizable and collapsible panes
2. User can select an agent and send messages with AI responses streaming token by token
3. User can click "Send to editor" on an AI chat message and see content inserted into the current chapter
4. Chat history, pane sizes, and collapse states are preserved across app restarts
5. Layout renders correctly at window widths from 1024px to 1920px+

### Phase 12: Setup Wizard

**Goal**: User can create a new book project through a conversational AI wizard that plans the story arc, chapter outline, themes, and characters before writing begins
**Depends on**: Phase 11 (uses chat infrastructure, gateway patterns, and 3-pane layout)
**Requirements**: WIZARD-01 through WIZARD-11
**Plans**: 4 plans

Plans:
- [x] 12-01: Wizard foundation — store, Zod schemas, prompts, IPC service, preload bridge, type flexibility
- [x] 12-02: Wizard UI — SetupWizard orchestrator, WizardChat with streaming, routing, ProjectSelector button
- [x] 12-03: Review + creation — WizardReview form, WizardPlanPreview, atomic createFromWizard IPC
- [x] 12-04: Integration — wire review/preview into wizard, resume logic

**Success Criteria:**
1. User can start a new book and enter a brain dump, then converse with AI that proposes arc, chapters, characters
2. User can review and edit the proposed plan before confirming project creation
3. On completion, project created with chapters, characters, and timeline pre-populated
4. User can quit mid-wizard, restart the app, and resume where they left off
5. User can skip the wizard entirely and use quick-create

## Milestone Summary

**Key Decisions:**
- react-resizable-panels v4.6.2 and @tiptap/markdown v3.19.0 as only new dependencies
- Separate Zustand stores per feature (chatPaneStore, wizardStore) following feedbackStore pattern
- Gateway session key namespacing for full isolation (chat, feedback, wizard, wizard-extract)
- pendingInsert in writingStore for decoupled chat-to-editor content flow
- JSONL append-only format for chat history (matches feedback log pattern)
- Wizard uses prompt-level JSON extraction (JSON in markdown code blocks + Zod validation)
- Atomic project creation with filesystem rollback on failure
- ProjectMeta.type changed from union to string for genre flexibility
- Resume detection with inline banner (not modal)

**Issues Resolved:**
- None (clean execution across both phases)

**Issues Deferred:**
- Pre-existing 115 TypeScript errors in Dashboard.tsx, ChatRoomView.tsx, VoiceChatPanel.tsx (unrelated to v2.1)
- ProjectSelector memoir-specific badge color check (should be generic for wizard-created genres)
- Session key format built inline in 4 places (could extract to shared utility)

**Technical Debt:**
- electron/main.ts remains a 7,000+ line monolith (tracked separately)
- preload namespace still `clawdbot` (cosmetic rename deferred)
