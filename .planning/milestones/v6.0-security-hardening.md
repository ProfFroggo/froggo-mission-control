# Milestone v6.0: Security Hardening

**Status:** ✅ SHIPPED 2026-03-07
**Phases:** 50–57
**Total Plans:** 9 (phases 52 + 53 absorbed into Phase 50 plans)

## Overview

Eliminated all critical and high security findings from the platform audit — command injection, path traversal, credential exposure, and input validation gaps. All fixes are defence-in-depth: agent ID validation applied at every route that touches the filesystem or spawns processes.

### Phase 50: agent-id-validation

**Goal**: Create a shared `validateAgentId()` utility enforcing `/^[a-z0-9][a-z0-9-_]*$/` and apply it to every route that uses agent ID in file path construction or process spawning.
**Depends on**: Phase 49
**Plans**: 2 plans

Plans:
- [x] 50-01: Core routes (spawn, stream, soul) — validateAgentId + command injection fix + hardcoded path removal
- [x] 50-02: All remaining routes (catalog, hire, avatar, kill, config, session, status, models)

**Details:**
- `src/lib/validateId.ts` created — exports `AGENT_ID_PATTERN = /^[a-z0-9][a-z0-9-_]*$/` and `validateAgentId(id: unknown): NextResponse | null`
- `spawn/route.ts`: replaced `execSync(\`bash "${scriptPath}" "${id}"\`)` with `spawnSync('bash', [scriptPath, id])` — eliminates command injection
- `soul/route.ts`: removed hardcoded `/Users/kevin.macarthur/git/mission-control-nextjs` — replaced with `process.cwd()`
- Full coverage: 10+ routes all guarded

### Phase 55: csp-security-headers

**Goal**: Add security response headers to all routes via `next.config.js` `headers()` config.
**Depends on**: Phase 53
**Plans**: 1 plan

Plans:
- [x] 55-01: Add security headers to next.config.js

**Details:**
- `X-Frame-Options: DENY` — blocks clickjacking
- `X-Content-Type-Options: nosniff` — prevents MIME sniffing
- `Referrer-Policy: strict-origin-when-cross-origin` — limits referrer leakage

## Milestone Summary

**Key Decisions:**

- `validateAgentId` shared utility pattern — one function, applied everywhere, returns `NextResponse | null`
- `spawnSync` with args array (not execSync with interpolated string) — eliminates shell injection class
- `ALLOWED_ROOTS` allowlist pattern for file reads (not denylist) — safer default
- Gemini key stored in DB settings table, never in localStorage or VITE_ env vars
- CSP via `next.config.js headers()` (not middleware) — simpler, no edge runtime needed
- Input length caps are pragmatic (not RFC-strict): 500/5000/100/255/50KB chosen for DoS prevention
- Static smoke test checks (grep-based) — no running server required, catches regressions in CI

**Issues Resolved:**

- Command injection in spawn route (execSync → spawnSync)
- Path traversal in library view action
- Path traversal in files/read route
- Hardcoded absolute path in soul route
- Gemini API key exposure in localStorage
- Missing input validation on task/hire/library/soul/modules routes
- No security headers on HTTP responses

**Issues Deferred:**

- AgentPanel.tsx TypeScript error (pre-existing, unrelated to security)
- 531 `window.clawdbot` refs via bridge.ts (carry-over from v1.0)

**Technical Debt Incurred:**

- Phase 54-01 SUMMARY.md was not written (work is committed and verified via git log)
- Phases 52 and 53 SUMMARY.md were not written (work absorbed into Phase 50-01)
