# Milestone v4.0: Agent & Module Library

**Status:** ✅ SHIPPED 2026-03-06
**Phases:** 31–39 (including 33.1 inserted)
**Total Plans:** 12

## Overview

Any team member can browse a catalog of agents and modules, hire/install with a wizard-guided flow backed by HR and Coder agents, and have their Mission Control instance automatically configured — without touching a CLI. Full lifecycle management (fire, uninstall, re-hire) and onboarding role presets round out the self-service experience.

## Milestone Summary

**Key Decisions:**

- Catalog tables additive (separate from `agents`/`module_state`) — no migration risk
- ON CONFLICT preserves hire state; manifests own metadata only
- Non-destructive fire: workspace rename not delete (preserves work history)
- Core module guard at API layer (not just UI) for defense in depth
- HR chat uses `claude --print` with full self-contained prompt (not streaming)
- MEMORY.md idempotent on re-hire (preserves accumulated agent memory)
- `Promise.allSettled` for role preset installs (partial failure OK)
- Haiku for HR chat (trivial conversational task), Sonnet for coding agents

**Issues Resolved:**

- `POST /api/agents/hr/stream` endpoint missing (HRAgentCreationModal broken) — fixed in 33.1
- TypeScript error: `title` prop on LucideIcon — removed from AgentLibraryPanel
- Fragment not closed in AgentPanel — fixed via targeted Edit
- Backslash-escape bug in smoke test `\[id\]` file paths — fixed in Phase 39

**Issues Deferred:**

- Streaming HR agent for workspace generation (currently step-progress UI, no live stream)
- Role preset install: saved to DB for future catalog recommendations (not implemented)
- Agent re-hire UX: full config preservation on re-hire (CLAUDE.md/SOUL.md overwrite on re-hire)

**Technical Debt Incurred:**

- ModuleInstallModal calls both `moduleApi.install()` AND `catalogApi.setModuleInstalled()` (belt-and-suspenders; could be consolidated into one call)
