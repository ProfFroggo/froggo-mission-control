# Plan 02-01: Remove Glassmorphism from Content Cards

<objective>
Remove all `backdrop-blur` and semi-transparent backgrounds from content cards in
`DashboardRedesigned.tsx`. Six panel wrappers use `bg-mission-control-surface/80 backdrop-blur-xl`
— glassmorphism on data cards is a visual anti-pattern that reads as AI-generated. Replace with
solid `bg-mission-control-surface` using standard border.

Also remove the outer page-level gradient backgrounds (line 130, 132) — they create a hazy visual
layer that serves no purpose. Replace with flat `bg-mission-control-bg`.

Also clean up the dead `.card-glass` glassmorphism CSS class that remains in component-patterns.css
and glass-theme.css — now that glass-theme.css is imported, this class is active and affects
anything using it.
</objective>

<execution_context>
@src/components/DashboardRedesigned.tsx
@src/component-patterns.css
@src/glass-theme.css
</execution_context>

**Replacement pattern** (remove /80 opacity and backdrop-blur; use full border opacity):
```
bg-mission-control-surface rounded-2xl border border-mission-control-border overflow-hidden
```
Drop `shadow-xl/shadow-2xl` too — the `glass-card` definition in component-patterns already
provides box-shadow via `var(--shadow)`.

**Page-level gradient wrappers:**
- Line 130: `bg-gradient-to-b from-mission-control-bg to-mission-control-surface` → `bg-mission-control-bg`
- Line 132: `bg-gradient-to-br from-mission-control-surface via-mission-control-bg to-mission-control-surface border-b border-mission-control-border/50` → `bg-mission-control-surface border-b border-mission-control-border`
- Line 365: header section `bg-gradient-to-r from-mission-control-surface to-mission-control-bg` → remove this class (the section already has the outer surface bg)
