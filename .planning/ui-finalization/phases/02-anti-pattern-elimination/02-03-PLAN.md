**1. `animate-gradient-x` — dead utility class (2 instances in DashboardRedesigned.tsx):**
- Line 134: `animate-gradient-x` on animated gradient overlay div
- Line 226: `animate-gradient-x` on another overlay div
The class `animate-gradient-x` is not defined in `tailwind.config.js`. These elements produce no
animation. Remove the `animate-gradient-x` class from both divs (the elements themselves can stay
or be simplified — they're decorative overlays).

**2. Hardcoded shadow palette colors in `DashboardRedesigned.tsx` (3 instances):**
- Line 340: `shadow-green-400/50` on status indicator dot
- Line 410: `shadow-purple-400/50` on review status dot
- Line 411: `shadow-blue-400/50` on in-progress status dot

These use Tailwind palette colors not from the design token system. Replacements:
- `shadow-green-400/50` → remove entirely (status dot has `animate-pulse` which provides the effect; glow shadow is gratuitous, deferred to Phase 7 proper delight moments where it'll be done with CSS custom property approach)
- `shadow-purple-400/50` → remove
- `shadow-blue-400/50` → remove

**3. `AgentPanel.tsx:65` — inline style `style={{ backgroundColor: themeColor + '22' }}`:**
Context: This is the fallback avatar when no image exists — shows agent initials on a color background.
`themeColor` is a hex value like `#a855f7` (violet). Appending `'22'` creates `#a855f722` which
is hex with ~13% alpha opacity. This is a fragile pattern — it assumes themeColor is always
6-char hex (breaks with CSS vars, rgb() strings, etc.).

Check what `themeColor` is in the component context:
```bash
grep -n "themeColor" src/components/AgentPanel.tsx | head -20
```
Fix: Convert to CSS with `color-mix(in srgb, ${themeColor} 13%, transparent)` OR
if `themeColor` is always a hex from a fixed palette, keep as-is but document it clearly.
If the component accepts a CSS color token string: use inline style with `opacity` on the
container and keep background as the solid color.

**4. TSX component green rgba (6 total occurrences, 3 files):**
Run grep to get exact lines:
```bash
grep -n "rgba(34, 197, 94" src/components/Kanban.tsx src/components/ModuleDependencyGraph.tsx src/components/CampaignTimelineView.tsx
```
Convert using same logic as Phase 1:
- Accent usage → `color-mix(in srgb, var(--mission-control-accent) N%, transparent)`
- Semantic success → `color-mix(in srgb, var(--color-success) N%, transparent)`
- Determine context (is it a success indicator or decorative?)

Note: In TSX files, inline CSS properties use JS string syntax:
`"rgba(34, 197, 94, 0.1)"` → `"color-mix(in srgb, var(--mission-control-accent) 10%, transparent)"`
OR if as Tailwind arbitrary: `bg-[color-mix(in_srgb,var(--mission-control-accent)_10%,transparent)]`
