# Plan 01-03: Eliminate All Hardcoded Green rgba Values

<objective>
Replace all 28 hardcoded `rgba(34, 197, 94, ...)` occurrences across CSS files with
`color-mix(in srgb, var(--mission-control-accent) N%, transparent)` equivalents. The accent color
is violet (`accentColor="violet"`). Any hardcoded green focus ring, glow, or highlight that fires
on non-semantic elements (i.e., not approve/deny pairs) breaks the violet design system.

Critical: `glass-theme.css:222-226` applies green focus ring with `!important` to ALL inputs —
this overrides the Radix Themes focus system and must be removed.

NOTE: This plan covers CSS files only. TSX component green rgba instances (Kanban.tsx,
ModuleDependencyGraph.tsx, CampaignTimelineView.tsx) are addressed in Phase 2 as part of the
broader anti-pattern elimination.
</objective>

**Special case — semantic success tokens in `design-tokens.css`:**
Lines 100-101: `--color-success-bg` and `--color-success-border` are SEMANTIC green (success state),
not accent. These should STAY green but use the semantic token variable:
```css
/* Keep green but use token var for consistency */
--color-success-bg: color-mix(in srgb, var(--color-success) 10%, transparent);
--color-success-border: color-mix(in srgb, var(--color-success) 30%, transparent);
```
`--color-success` is already defined as `#22c55e` on line 99 — so this is self-referential and correct.

**Special case — shadow-glow tokens in `design-tokens.css:178-179`:**
These are currently green-tinted. They should be accent-colored:
```css
--shadow-glow: 0 0 20px color-mix(in srgb, var(--mission-control-accent) 30%, transparent);
--shadow-glow-lg: 0 0 40px color-mix(in srgb, var(--mission-control-accent) 40%, transparent);
```

**Critical case — `glass-theme.css:219-226` INPUT FOCUS RING:**
```css
input:not(.rt-TextFieldInput):focus,
textarea:not(.rt-TextAreaInput):focus,
select:not(.rt-SelectTrigger):focus {
  border-color: rgba(34, 197, 94, 0.5) !important;     /* ← REMOVE !important, use accent */
  box-shadow: 0 0 0 2px rgba(34, 197, 94, 0.10) !important;  /* ← REMOVE !important, use accent */
  background: rgba(255, 255, 255, 0.06) !important;
  outline: none !important;
}
```
Replace with:
```css
input:not(.rt-TextFieldInput):focus,
textarea:not(.rt-TextAreaInput):focus,
select:not(.rt-SelectTrigger):focus {
  border-color: var(--mission-control-accent);
  box-shadow: 0 0 0 2px color-mix(in srgb, var(--mission-control-accent) 10%, transparent);
  background: rgba(255, 255, 255, 0.06);
  outline: none;
}
```
Removing `!important` is safe here because Radix Themes injects focus styles scoped to `.rt-*`
elements (already excluded via `:not(.rt-TextFieldInput)` etc). Without `!important`, the Radix
focus styles on `.rt-*` elements will not be overridden.
