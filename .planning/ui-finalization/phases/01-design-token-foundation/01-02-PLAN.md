# Plan 01-02: Fix Base Font Size

<objective>
Fix `--mission-control-font-size: 14px` → `1rem` in `src/design-tokens.css:72`. The body element
uses this token directly (`font-size: var(--mission-control-font-size)`). Using `14px` instead of
`1rem` breaks user browser font-size preferences (accessibility violation) and fails WCAG SC 1.4.4
(Resize Text). The value `1rem` equates to 16px at default browser settings — the correct baseline.
Also audit the font-size slider in settings to ensure the UI min/max bounds are correct.
</objective>

<execution_context>
@src/design-tokens.css
@src/index.css
</execution_context>

<context>
## Discovery Findings

**`src/design-tokens.css:72`:**
```css
--mission-control-font-size: 14px;
```
Must become: `1rem`

**`src/index.css:40`:**
```css
font-size: var(--mission-control-font-size);
```
This is correct — body consumes the token. No change needed here.

**Settings panel font-size slider:** `EnhancedSettingsPanel.tsx` has a font-size control. The
roadmap notes line 227 as a potential min-bound fix. After changing from 14px to 1rem, the slider
should still allow users to set 14px (0.875rem) as minimum if they prefer compact UI — but the
*default* must be 1rem.

**Visual impact:** All text in the app will appear slightly larger (16px vs 14px at default browser
zoom). This is intentional — 14px is below the WCAG minimum recommendation for body text.
Typography hierarchy remains intact since all heading/caption sizes are defined in rem relative to
the font root.
</context>
