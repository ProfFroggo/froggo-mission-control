# Plan 01-05: Fix Placeholder Contrast

<objective>
Remove the double-opacity on placeholder text in `src/forms.css:107-110`. The current rule applies
`opacity: 0.6` to a color that is already `var(--mission-control-text-dim)` (gray-9 in Radix scale,
already a dim/secondary color). Stacking two levels of opacity reduces contrast below WCAG AA
minimum (4.5:1) in dark mode. Fix by removing the `opacity` property — let `text-dim` carry the
semantic weight alone.

Also raise `--leading-normal` from `1.5` to `1.55` in `design-tokens.css` for slightly improved
dark-mode body text legibility (tighter line spacing causes visual crowding at 16px).
</objective>

<execution_context>
@src/forms.css
@src/design-tokens.css
</execution_context>

<context>
## Discovery Findings

**`src/forms.css:104-110`:**
```css
/* Placeholder text */
.mission-control-input::placeholder,
input:not(.unstyled)::placeholder,
textarea:not(.unstyled)::placeholder {
  color: var(--mission-control-text-dim);
  opacity: 0.6;
}
```

`var(--mission-control-text-dim)` resolves to `var(--gray-9)` (defined in `design-tokens.css`
`.radix-themes` bridge). Radix gray-9 in dark mode is approximately `#868e96` (rgb 134,142,150).
On `--color-background` (dark: ~rgb 17,17,17), gray-9 alone achieves ~4.6:1 contrast (barely AA).
After `opacity: 0.6`, effective color is ~rgba(134,142,150,0.6) ≈ `#5e6570` on dark background —
approximately 3.1:1 contrast ratio. This FAILS WCAG AA.

**Fix**: Remove `opacity: 0.6`. The `text-dim` color alone provides sufficient visual distinction
between placeholder and filled text without failing contrast requirements.

**`src/design-tokens.css:50` (--leading-normal):**
```css
--leading-normal: 1.5;
```
Change to `1.55` — minor adjustment that improves readability at 16px body text in dark mode by
slightly increasing the breathing room between lines. WCAG recommends at least 1.5 for body text;
1.55 gives a small margin above the minimum.
</context>
