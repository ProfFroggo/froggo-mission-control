# Plan 01-06: Remove ease-bounce + Consolidate Button CSS Systems

<objective>
Two cleanup tasks that complete the token foundation:

1. **Remove `--ease-bounce`**: The `cubic-bezier(0.68, -0.55, 0.265, 1.55)` bounce easing is
   present in `design-system/tokens.css` (being deleted in 01-01). Ensure it does NOT exist in
   `design-tokens.css`. Add `--ease-quint-out: cubic-bezier(0.22, 1, 0.36, 1)` as the canonical
   premium easing for all UI transitions. This easing is used by Stripe, Linear, Vercel — fast start,
   smooth deceleration, no overshoot.

2. **Consolidate 3 competing button CSS systems**: There are three separate button style systems:
   - `.btn` in `index.css` (base reset)
   - `.btn-base` in `component-patterns.css` (secondary system)
   - `.mission-control-btn-*` in `forms.css` (explicit opt-in system)
   These compete and create confusion about which to use. Consolidate into one canonical system
   in `component-patterns.css`.
</objective>

**Canonical button system design** (for `component-patterns.css`):
The app uses Radix `<Button>` for primary/secondary actions. Raw `<button>` elements are used for:
- Nav tabs (`border-b-2` style)
- Segment controls (bordered container style)

The canonical `.mission-control-btn` class is an opt-in for raw `<button>` elements that need
base styling (cursor, font-inherit, focus ring). Keep it lean:
```css
/* Opt-in base for raw <button> elements (nav tabs, segment controls use their own Tailwind) */
.mission-control-btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: var(--space-inline);
  cursor: pointer;
  font: inherit;
  border: none;
  background: none;
  padding: 0;
  transition: var(--transition-base);
}
.mission-control-btn:focus-visible {
  outline: var(--focus-ring-width) solid var(--color-focus-ring);
  outline-offset: var(--focus-ring-offset);
}
.mission-control-btn:disabled {
  cursor: not-allowed;
  opacity: 0.5;
}
```

**Strategy for consolidation:**
1. Identify what `.btn` in `index.css` and `.btn-base` in `component-patterns.css` define
2. If they add meaningful base styles → merge into `component-patterns.css` `.mission-control-btn`
3. If `.btn` is used by Tailwind's `@layer base` reset → leave it in `index.css`
4. Remove redundant definitions; update component usages if class names change
