## TECHNICAL ARCHITECTURE

### Core Stack
- **Frontend:** PWA (Progressive Web App) + Next.js microfrontend (Module Federation)
- **Backend:** Express.js (API), Node.js event-driven
- **Database:** Aurora PostgreSQL 15.12 (via SSH tunnel/Bastion)
- **Message Queue:** RabbitMQ (for real-time trade event processing)
- **ORM:** Prisma
- **Authentication:** Privy (wallet, email, social)
- **Routing:** Carbium (GPU-based DEX aggregator)
- **Data providers:** BirdEye (token data), Birdeye (real-time streaming)

### Colosseum Technical Components
- **45+ domain entities**
- **47 repository interfaces**
- **157+ application handlers**
- **37 user + 80+ admin API endpoints**
- **50+ database tables**
- **6 Reactor event handlers**
- **8+ Cron jobs**

### Carbium Routing Engine (CQ1)
- **Data format:** Raw binary buffers (vs. JSON)
- **Pathfinding:** GPU-based parallel processing (millions of routes)
- **Quote latency:** <1ms (vs. 50–500ms competitors)
- **Transaction execution:** <700ms
- **Positive slippage:** Returned to users on select trades (+0.02% to +0.08%)

### Real-Time Data Architecture (2026 Upgrade)
**Onchain CC 2.0 components:**
- **Multiplexer:** Aggregates websocket connections to prevent rate limit overages
- **BirdEye integration:** Real-time token discovery streaming
- **Carbium gRPC:** Blockchain data querying
- **GeckoTerminal:** Liquidity pool data for pre-graduation tokens
- **1-second timeframe charts:** For high-velocity small-cap trading

### Landing Page Infrastructure
- **Hosting:** AWS S3 + CloudFront CDN
- **Build:** No-framework (HTML5/CSS/JS)
- **CI/CD:** GitHub Actions
- **Analytics:** Mixpanel (EU data center)
- **i18n:** Client-side translation (EN, ES, PT, ZH)

## STRATEGIC INSIGHTS & OPERATIONAL PHILOSOPHY

### What Worked
1. **Community-led growth:** 30–50x more efficient than paid KOL model
2. **Points over cash:** Delays rewards, filters farmers, incentivizes traders
3. **Discord coordination:** Self-organizing, language-specific subcommunities
4. **Fee transparency:** 0.15%–0.2% is a structural moat, not a marketing claim
5. **Gamification retention:** Leaderboards + streaks + loot boxes drive daily habits
6. **Fast pivots:** Decision cycle measured in days, not quarters

### What Failed
1. **KOL contracts:** Expensive, poor retention, difficult to scale
2. **LATAM retail positioning:** Not the product's actual user base
3. **CLM delays:** Q4 visibility spike without follow-up path = lost conversion
4. **Perps timeline:** 6 months of "coming soon" parked whale capital
5. **Invite code friction:** Scarcity works during viral events; hurts cold acquisition

### Operating at Scale
- **Current team:** 2 people (Kevin + Tayler) running strategy, community, socials, operations
- **Burnout signal:** "12–16 hour days for six months" flagged in Jan 30 approval document
- **Needed:** Solana Ecosystem Lead (expected 2x volume multiplier), additional capacity
- **Leverage:** n8n automation, Claude agents, Supabase for shared memory

### The Unplanned Winners
- **Discord:** Not in original GTM plan; emerged as crisis response; became primary moat
- **United States:** Mexico-first plan; US ended up 12:1 user ratio vs. Mexico
- **Fee-led positioning:** Competitive pricing not mentioned before Dec 2025; primary differentiator by Jan 30
